# 售前方案训练营 · 学习路径总入口

> 面向**纯新人**(转岗、应届)的售前方案能力养成课程。
> 目标:把"没写过方案、不懂售前"的新人,带到"能独立驾驭 Agent 团队、交付一份打得出去的技术方案"。

---

## 一、这套教材是什么

它不是"方案写作技巧合集",而是一套 **学—练—产 闭环训练营**:

- **学**:搞懂售前是什么、方案为什么是"销售武器"、一份好方案靠哪 16 招撑起来。
- **练**:用真实标杆案例逐段拆解,看招式怎么落地;再做模拟客户演练。
- **产**:用 Claude Code 驱动本仓库的 **Agent 团队**,从一份客户调研笔记,真打一份完整方案出来。

### 公司两大主营领域(教材覆盖)

| 领域 | 标杆案例 | 强在哪 |
|---|---|---|
| 智能座舱 / 展陈 | 智能太空舱方案 | 叙事与定位(升维、记忆锚点、证据) |
| **船舶智能化 / 智能航运** | 元启晨星船舶方案 | 文档骨架与拿单(决策问题弧、信任前置、分步推进) |

两份标杆互补,招式卡片(02)同时从两者提炼。

### 核心心法:Agent 团队 = 16 招的"专家化身"

本仓库已有一支覆盖售前全生命周期的角色化 Agent 团队。**你不需要把 16 招都练成肌肉记忆**——你要练的是更值钱的能力:**什么时候该叫哪个专家、怎么给它喂料、怎么验收它的产出。**

| 链路环节 | 主导 Agent | 对应招式 | 标杆案例里的体现 |
|---|---|---|---|
| ① 挖痛点 / 技术发现 | 🛠️ Sales Engineer | 痛点回指、技术翻译、痛点收敛 | 太空舱 2.4 升级理由;船舶 P08 表象→根因→后果 |
| ① 干系人 / 政治地图 | 🗺️ Account Strategist | 受众分层、一材多版 | 太空舱 1.4 三类受众;船舶 P06A/B/C 三版痛点 |
| ② 定位 / 赢主题 / 叙事 | 🏹 Proposal Strategist | 升维叙事、记忆锚点、证据先亮、信任前置、决策问题弧 | 太空舱双重定位/十二个「可」;船舶 P02 认证前置、6章决策弧 |
| ③ 技术主体 / 架构 | 🛠️ SE + 📐 UX Architect + 🧭 PM | 架构地图、价值对比矩阵、分级切入 | 太空舱一舱一域一环境三端;船舶四维对比、P10 分级切入 |
| ④ 计划 / 风险 / 交付 | 🐑 Project Shepherd | 验收颗粒度、开放项闭环、小步快跑+退出 | 太空舱 15.3 关键路径;船舶 P25 三步走+退出机制 |
| ⑤ 拿单 / 答辩 / 复盘 | 🏋️ Sales Coach | 临门一脚 CTA、全盘自检 | 船舶 P26/P27 政策申报+预约演示 |

## 三、三条学习路径(按你的起点选)

| 你是谁 | 路线 | 预期周期 |
|---|---|---|
| **应届 / 完全没接触过售前** | 01 → 02 → 案例批注 → 03 → 练习 E1/E2 → 04 → 实战 E3 | 2–3 周 |
| **转岗(懂技术,不懂销售)** | 01(快读)→ 02 → 03 → 04 → 实战 E3 | 1 周 |
| **要立刻上手干活** | 03 Agent 手册 → 模板库 → 实战 E3,边干边回查 02 | 2–3 天 |

## 四、学完你应该能做到

- [ ] 听到一段客户需求,能立刻分辨「这是痛点还是表面要求」
- [ ] 把一句技术参数,翻译成客户听得懂的业务价值
- [ ] 说清楚一份方案该有哪些部分、每部分谁来写、先写哪个
- [ ] 独立调用 Agent 团队,从调研笔记产出方案初稿
- [ ] 拿一份草稿对着「全盘自检」过一遍,知道它能不能打出去

---

## 五、怎么用 Claude Code 配合本教材

1. **学的时候**:让任意 Agent 当陪练。例如读完 02 后,把一段平庸的功能描述丢给 🛠️ Sales Engineer,看它怎么翻译成业务价值。
2. **练的时候**:用 🏹 Proposal Strategist 给你的练习产出打分。
3. **产的时候**:按 `04-流程篇` 的剧本,依次召唤各 Agent,串成一条产线。
4. **出片的时候**:内容定稿后,按 `06-排版篇` 把 Markdown 一键渲染成专业 PDF 文档或答辩 deck——**版面是信任的一部分**。

> 详细召唤方式见 `03-Agent团队使用手册.md`;出片选型与命令见 `06-排版篇-把方案变好看.md`。
