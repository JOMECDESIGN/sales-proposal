# 流程篇 · 从挖需求到拿单

> 前面学了招式(02)和工具(03 Agent 团队)。本篇把它们串成一条**完整作战流程**:
> 一个真实项目从"客户冒头"到"拿下第一步",分五段,每段告诉你**干什么、叫谁、用哪招、交付什么、怎么算过关**。
>
> 这也是"产"阶段实战(`练习/E3`)的剧本。

---

## 全流程一图流

```
① 挖需求      ② 定方向      ③ 搭主体      ④ 出计划      ⑤ 拿单
  痛点+受众  →  赢主题+架构  →  技术+案例  →  交付+风险  →  自检+CTA
  🛠️            🏹 🧭          🛠️ 📐 🧭        🐑 🧭         🏋️
```

> 铁律:**不要跳段。** 新人最大的坑是跳过①②直接写③。没挖透痛点、没定好方向就堆技术,产出必然是"漂亮的空话"。

---

## 第①段 · 挖需求(痛点 + 受众)

**目标**:搞清楚"客户真正的麻烦是什么"和"谁会看这份方案"。

**怎么做**:
1. 收集一切原始素材:招标文件、会议纪要、客户原话、现状描述。
2. 召唤 🛠️ **Sales Engineer**:挖出真痛点并把零散痛点**收敛到根因**(招 12);同时画干系人地图、分出决策者/技术/采购三类受众(招 3 的发现部分)。
3. 各角色的**阅读动线设计**(招 3 的另一半)留到第②段,由 🏹 **Proposal Strategist** 结合赢主题一并完成。

**交付物**:痛点清单(表象→根因→后果)+ 受众分层表。

**过关标准**:
- [ ] 每个痛点都能追溯到客户的原话或事实,不是你脑补的。
- [ ] 痛点收敛到了一个根本原因,而不是十个并列项。
- [ ] 三类受众各自关心什么,写清楚了。

---

## 第②段 · 定方向(赢主题 + 架构)

**目标**:定下"我们靠什么赢"和"方案怎么搭"。

**怎么做**:
1. 把①的产出喂给 🏹 **Proposal Strategist**:提炼 3-5 个赢主题(招 5 升维)、设计决策问题弧(招 16)、选出最强信任锚点前置(招 11)。
2. 需要平台/产品体系叙事兜底时,叫 🧭 **Product Manager**。
3. 先写**执行摘要**——它逼你把论点想清楚,再展开细节。

**交付物**:赢主题矩阵 + 方案架构蓝图(6 章决策弧)+ 一页执行摘要 + 记忆锚点。

**过关标准**:
- [ ] 赢主题换个客户名就不成立(说明够具体)。
- [ ] 每章对应一个决策问题,章章承接。
- [ ] 有一个能被复述的记忆锚点。

---

## 第③段 · 搭主体(技术 + 案例)

**目标**:把方案的技术主体和可信度填实。

**怎么做**:
1. 🛠️ **Sales Engineer** + 📐 **UX Architect** + 🧭 **PM**:搭技术主体,用一句口诀定架构主线(招 7),每条能力回指痛点(招 1)、翻译成价值(招 2)。
2. 做**价值对比矩阵**(招 8):我们 vs 现状/竞品,只摆事实不贬低。
3. 组织**已验证能力**:用"挑战→核心技术→成果"的标准结构写案例(招 6)。

**交付物**:技术主体章节 + 架构图 + 价值对比矩阵 + 案例集。

**过关标准**:
- [ ] 架构有一条外行也看懂的主线。
- [ ] 每个主张都配了证据(数字/案例/方法名)。
- [ ] 没有一句"业界领先""强大"之类的空话。

---

## 第④段 · 出计划(交付 + 风险)

**目标**:回答"你们交得了货吗",并把风险摊开、可控。

**怎么做**:
1. 🐑 **Project Shepherd**:做交付物与验收表(招 9)、关键路径日历、待定项与风险登记(招 10)。
2. 🧭 **Product Manager**(给分级产品依据)+ 🏹 **Strategist**(②已定的分级定位):给**分级切入**入口(招 13)——把方案拆成小/中/大三档起点,主动推荐"以您的情况,建议先从 X 切入",先把首次决策门槛降下来。
3. 在分级入口之上,设计**三步走合作路径 + 退出机制**(招 14),把第一步做到尽量轻。

**交付物**:交付物与验收表 + 关键路径 + 待定项/风险登记 + 分级切入入口 + 三步合作路径。

**过关标准**:
- [ ] 每个交付物有"怎么算做到了"的验收标准。
- [ ] 待定项都有负责人和闭环时间。
- [ ] 给了小/中/大分级入口,客户能选轻档起步。
- [ ] 第一步足够轻,且写明了退出机制。

---

## 第⑤段 · 拿单(自检 + CTA)

**目标**:把方案打磨到能打出去,并给客户一个能立刻迈出去的下一步。

**怎么做**:
1. 🏋️ **Sales Coach**:当挑剔评委做全盘自检——挑空话、挑无证据的主张、挑没回指痛点的段落。
2. 逐条改完,再设计**临门一脚 CTA**(招 15):一个低风险下一步(预约现场演示/免费现状诊断)+ 可执行承诺(如 48 小时回复)。
3. 顺手帮客户**向上汇报**:把项目包装成可申报的政策/预算素材。

**交付物**:终稿 + 自检报告 + 明确 CTA。

**过关标准**:
- [ ] Coach 指出的问题逐条改了。
- [ ] 结尾有具体、低风险、有时限的下一步。
- [ ] 经办人拿着这份方案,有东西能回去给领导讲。
