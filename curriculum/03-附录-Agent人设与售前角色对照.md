# 附录 · Agent 人设与售前角色对照(喂料前必读)

> **为什么要有这份对照**:本仓库的 6 个 Agent 定义文件(根目录的 `*.md`)是**通用、英文、行业最佳实践**的专业人设;而 `curriculum/` 教材是**中文、公司自有(太空舱/船舶)**的售前产线说明书。
> 两层并不天然重合——有的人设原本是为**别的场景**写的。直接 `@` 根文件、不框定场景,产出就会"跑偏"(带 B2B-SaaS、CSS、售后续费的腔调)。
> **这份对照表告诉你:每个 Agent 的"真身"是什么、在售前产线里干什么、喂料时要怎么一句话框定它。**

---

## 一、两层结构(先理解再用)

| 层 | 在哪 | 性质 | 你怎么对待 |
|---|---|---|---|
| **人设层** | 根目录 6 个 `.md` | 通用专家大脑,能力强但不针对"你这单售前" | 当"素材/发动机",**用前先框定场景** |
| **教材层** | `curriculum/` | 把通用专家编排进售前产线(16 招 + 五段流程) | 当"操作规范",照它的喂料示例喂 |

> 一句话:**人设给能力,教材给岗位。** 二者之间的"框定话术",就是这份附录。

---

## 二、逐个 Agent:真身 → 售前角色 → 框定话术

### 🏹 Proposal Strategist · 方案策略师 —— **对口度最高,可直接用**
- **人设真身**:纯正提案策略师(赢主题、三幕叙事、执行摘要、Shipley/color-team 评审)。
- **售前角色**:招 4/5/6/11/13/16 + 招 3 的受众阅读动线。定赢主题、搭决策弧、写执行摘要。
- **框定话术**:基本无需框定。直接给"痛点清单 + 受众分层 + 差异点",让它出赢主题与叙事弧即可。

### 🛠️ Sales Engineer · 售前工程师 —— **对口,可直接用**
- **人设真身**:售前工程师(技术发现、Demo、POC、FIA 竞争卡)。本就是售前角色。
- **售前角色**:招 1/2/8/12 + 招 3 的**干系人地图/三类受众发现**(原 Account Strategist 的活已并入这里)。
- **框定话术**:无需特别框定;要它画干系人地图时,明确"谁拍板/谁评估/谁付钱"三类即可。

### 🐑 Project Shepherd · 项目护航者 —— **对口,可直接用**
- **人设真身**:通用跨职能项目经理(章程、关键路径、风险、状态报告)。
- **售前角色**:招 9/10/14。交付物与验收表、关键路径、待定项/风险、三步走合作路径(**含商务路径**,原 Account Strategist 的商务部分已并入)。
- **框定话术**:提醒它产出是"**给客户看的方案章节**",不是内部 PMO 报告;验收标准要可勾选。

### 🧭 Product Manager · 产品经理 —— ⚠️ **需框定:它是做产品的,不是写方案的**
- **人设真身**:通用 SaaS PM"Alex"(PRD、RICE、路线图、Sprint 健康度)。
- **售前角色**:支撑招 5 升维、招 7 架构地图、招 13 分级切入(给分级的产品依据)。只取"产品全景 / 演进路线 / 分级依据"。
- **框定话术**:
  > "**别写 PRD / 不排 Sprint。** 这是售前方案,帮我把产品讲成『一个整体、多个入口』的全景,给出平台演进路线,以及小/中/大分级切入各自适合什么客户。"

### 📐 UX Architect · UX 架构师 —— ⚠️ **需框定:它默认在讲 CSS/前端**
- **人设真身**:前端/CSS 架构师(设计系统、明暗主题切换、给开发交接)。已清理掉跨系统残留引用,但正文重心仍偏 CSS。
- **售前角色**:招 7 架构地图(可视化)、辅助招 2 技术翻译。只取"信息架构 / 层次主线 / 外行也懂的架构图"。
- **框定话术**:
  > "**先不要写 CSS / 主题切换。** 帮我把这套系统画成一张外行也看懂的架构图,要一条像『一舱一域一环境三端』那样的主线,每个图块对应方案正文一章。"

### 🏋️ Sales Coach · 销售教练 —— ⚠️ **需框定:把它的提问对准"方案",而非"销售业绩"**
- **人设真身**:销售团队教练(Pipeline review、MEDDPICC、预测纪律、带 rep)。
- **售前角色**:全盘自检(贯穿 16 招)+ 招 15 临门一脚 CTA。当"挑剔评委"挑空话、挑无证据、挑没回指痛点。
- **框定话术**:
  > "**不评我的业绩 / 不做 Pipeline review。** 把这份方案初稿当客户评标委员会来挑:哪里是空话?哪个主张没证据?哪里没回指痛点?结尾的下一步够不够具体?"

## 三、关于已删除的 🗺️ Account Strategist

原人设是**售后**扩张专家(land-and-expand、NRR、QBR、防流失),与"售前首单"错位最大,已**整体移除**。它原先承担的招式按下表重新归属:

| 原招式 | 现在归谁 |
|---|---|
| 招 3 受众分层 | 🛠️ SE(干系人地图/发现)+ 🏹 Strategist(阅读动线) |
| 招 13 分级切入 | 🏹 Strategist(定位)+ 🧭 PM(产品依据) |
| 招 14 小步快跑 + 退出(商务路径) | 🐑 Project Shepherd |
| 招 15 临门一脚 CTA | 🏋️ Sales Coach |

> 团队由 7 人精简为 **6 人**。干系人地图能力没有丢失,只是回归到最早接触客户的 🛠️ SE 手里——这也更贴合售前"边做技术发现边摸组织"的真实节奏。
