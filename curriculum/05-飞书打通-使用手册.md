# 飞书打通 · 使用手册

> 前四篇教你驾驭 Agent 团队产出方案;这一篇教你把方案产线**接进飞书**,让产出能协作、能播报、能看板化管理。
>
> 工程实现与脚本在仓库的 [`feishu/`](../feishu/) 目录,本篇是新人视角的"该让哪个 Agent、连飞书的什么能力、怎么连"。

---

## 一、三层一句话

| 层 | 你得到什么 | 上游(GitHub) |
|---|---|---|
| ① Agent 层 | 7 个 Agent **直接读写**飞书云文档/多维表格/消息 | 官方 [lark-openapi-mcp](https://github.com/larksuite/lark-openapi-mcp) |
| ② 文档同步层 | 方案/教材 **Markdown ↔ 云文档**双向无损 | 社区 [feishu-cli](https://github.com/riba2534/feishu-cli) |
| ③ 自动化层 | 状态**卡片播报** + 管道**多维表格看板** | 官方 [oapi-sdk-python](https://github.com/larksuite/oapi-sdk-python) |

装与配,照 [`feishu/README.md`](../feishu/README.md) 的"快速开始 5 步"。本篇只讲**怎么用**。

## 二、Agent → 飞书能力对照(核心)

每个 Agent 的产出,都有对应的飞书落点。记住这张表,就知道方案做到哪一步该往飞书推什么。

| Agent | 它的产出 | 落到飞书的哪里 | 走哪层 |
|---|---|---|---|
| 🛠️ Sales Engineer | 痛点清单、价值对比矩阵 | 多维表格(痛点/价值库) | ① MCP / ③ 脚本 |
| 🗺️ Account Strategist | 干系人分层、商务路径 | 多维表格(干系人地图) | ① MCP |
| 🏹 Proposal Strategist | 执行摘要、方案正文 | 云文档(给客户协作) | ② feishu-cli / ① MCP |
| 🧭 Product Manager | 产品全景、路线图 | 知识库(长期沉淀) | ② feishu-cli |
| 📐 UX Architect | 架构图(Mermaid) | 云文档(随 Markdown 无损导入) | ② feishu-cli |
| 🐑 Project Shepherd | 验收表、关键路径、风险 | 多维表格(管道看板)+ 日历 + 审批 | ③ 脚本 / ① MCP |
| 🏋️ Sales Coach | 自检结果、复盘 | 消息卡片(播报)+ 多维表格(复盘库) | ③ 脚本 |

> 经验法则:**文档类**产出走第②层(无损、留痕、进 Git);**结构化/状态类**产出走第③层(看板、播报);**对话中即时的零散操作**走第①层 MCP。

## 三、把飞书嵌进五段流程

对照 [`04-流程篇`](04-流程篇-从挖需求到拿单.md) 的五段,每段收尾顺手做一个飞书动作:

| 流程段 | Agent | 顺手做的飞书动作 |
|---|---|---|
| ① 挖需求 | 🛠️ SE · 🗺️ Account | `pipeline_base.py add` 建一单(阶段=挖需求);痛点/干系人入多维表格 |
| ② 赢主题 | 🏹 Strategist | 执行摘要 `doc import` 成云文档,链接发群征求意见 |
| ③ 技术主体 | 🛠️ SE · 📐 UX · 🧭 PM | 架构图随 Markdown 导入云文档;产品全景进知识库 |
| ④ 交付计划 | 🐑 Shepherd | 验收表 → 多维表格;关键路径 → 飞书日历;评审 → 审批流 |
| ⑤ 答辩自检 | 🏋️ Coach | `notify.py` 发绿/黄/红卡片播报;`set-stage` 推进该单 |

## 四、三个最常用动作(抄了就能用)

**A. 把一份方案推给客户在飞书上协作**
```bash
source feishu/.env
feishu-cli doc import 方案/示例项目-执行摘要.md --title "示例项目-执行摘要" --verbose
# 把返回的文档链接发群;客户批注后:
feishu-cli doc export <doc_id> -o 方案/示例项目-执行摘要.md --download-images
git diff   # 看客户改了啥,确认后 commit
```

**B. 方案定稿,播报 + 推进看板**
```bash
python feishu/03-automation/pipeline_base.py set-stage --record recXXXX --stage "答辩自检"
python feishu/03-automation/notify.py --title "示例项目 · 定稿" --stage "答辩自检" \
  --status green --owner 张三 --link "https://xxx.feishu.cn/docx/xxxxx"
```

**C. 让 Agent 自己干(第①层 MCP 已接好)**
> 对 🐑 Project Shepherd 说:"把《交付物与验收表》同步进飞书多维表格,每个交付物一条记录;完成后在群里发张卡片汇报。"
> Agent 会通过 MCP 工具直接落地,你不用敲命令。

## 五、新人最容易踩的三个坑

1. **机器人没进群就发消息** → 必然失败。先把应用机器人拉进目标群,`feishu-cli chat list` 取 `chat_id`。
2. **改了权限没发版** → 飞书开放平台改 scope 后,必须**发布新版本**才生效。
3. **把 `.env` / `sync-map.yaml` 提交上去** → 含凭证或具体 doc_id,已被 [`.gitignore`](../.gitignore) 屏蔽;别手动 `git add -f` 强加。
