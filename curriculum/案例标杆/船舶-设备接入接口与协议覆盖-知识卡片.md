## 一、第一性原则:先按"涉控/只读"分堆,再谈协议

从驾驶舱角度看要接入的设备,**不能笼统写"接入"**。送审时专家会立刻把设备按**是否涉控**重新分两堆,描述口径必须跟着分:

| 堆 | 设备 | 接入性质 | 口径(必须这样写) |
|---|---|---|---|
| **只读采集** | GNSS、AIS、罗经、计程/测深、雷达、ECDIS、VDR、CCTV、多数机舱遥测 | 旁路 / 隔离 / 只读 | 不加载既有总线,不替代法定功能 |
| **涉控 / 受控接口** | 自动舵、推进车令、联锁类 PLC | 经受控网关、分级控制 | "辅助建议 → 三道闸 → 人确认",执行权留在既有控制系统 |
| **法定取证 / 安全** | VDR/S-VDR、火警、进水检测 | 只读镜像 / 辅助 | 绝不可改写;只降误报、辅助定位,不替代 |

> **红线**:`自动舵 / 推进 / 联锁 PLC` 写成"接入"就会被审到。这些归第五章"分级控制 + 三道闸 + 人始终可接管"的口径,命令下发**不在采集链路**、而经受控网关。

## 三、4 大类协议映射(方案 §协议覆盖表 / 附录 B 同源)

写方案时协议覆盖按**四类现场接入**组织,三处(§协议表、附录 B、附录 D)必须**同源**,否则交叉对照会发现不自洽:

1. **航海数据接口** — NMEA 0183 / **NMEA 2000** / **IEC 61162 系列**
   - NMEA 2000 即 **IEC 61162-3**;网络型按 **IEC 61162-450**。
   - 计程 ≠ 测深(两类设备、量纲不同);罗经分磁/陀螺/卫星;GNSS 写多星座并显名 **北斗 BDS**(信创加分)。
2. **工业总线** — Modbus TCP·RTU / CAN(**SLCAN·J1939**) / **Profinet** / **EtherNet·IP** / **OPC UA**
   - 主机·推进多走 **CAN J1939**;船厂自动化常见工业以太网与 OPC UA。
3. **视频接口** — **RTSP / ONVIF**(网络视频) / SDI / HDMI(模拟数字视频)
   - 仅辅助值守与态势呈现,**非法定瞭望**。
4. **硬接线 I/O** — 模拟量 AI·AO / 开关量 DI·DO / **RS-422·485**
   - 适配无智能协议的传感器与干接点;模拟量按量程标定,开关量按常开/常闭定义。

> 法定航海与安全设备(ECDIS/雷达/VDR/火警)一律**旁路、隔离、只读**接入,**不替代其法定功能**——对应 IEC 61174 / 62388 / 62288 与 SOLAS V/22。

## 四、术语精确化(船检会逐字看)

- **GPS → GNSS**:实船多星座,务必体现北斗 **BDS**。
- **计程仪/测深仪是两类设备**:航速(kn,对水/对地)与水深(m),信号源与量纲不同,附录 D 分行。
- **罗经分型**:磁罗经 / 陀螺罗经 / 卫星罗经;航向 HDT/THS 与艏摇率 ROT 是不同语句。
- **摄像头写 CCTV/视频监控**,注明非法定、仅辅助值守,避免与法定瞭望混淆。
- 附录 D 五列闭环:**设备 — 信号点 — 协议/接口 — 量纲 — 隔离方式(只读/旁路/隔离/受控)**,量纲与点性不可缺。

## 五、落到方案里的三处一致性要求

| 位置 | 内容 | 易错点 |
|---|---|---|
| § 协议覆盖表 | 4 大类协议 + 典型设备 + 说明 | 只写 NMEA/CAN/Modbus 三类(漏 2000/61162/J1939/RTSP/硬接线) |
| 附录 B 对外接口概览 | 物理口 + **协议维度** | 只有物理口、缺协议;完整 ICD 随详设提供 |
| 附录 D 设备接入点表 | **12 类设备全进表** + 五列 | 只列 5 行示例,比正文还少,显得"没说全" |

> 一句话:**接入面要够全(12 类全进附录 D),协议要够准(4 大类显名标准号),边界要分堆(涉控/只读两栏),三处同源。**
