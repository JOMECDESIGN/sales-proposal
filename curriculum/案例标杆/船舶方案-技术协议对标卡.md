# 标杆案例 ②-B · 船舶智能化方案(技术协议 / 智控系统集成)

> **业务领域**:船舶智能化 / 智能航运(**运营域**)——与 `船舶方案-架构思路.md` 是**同一领域的两个轴**:那份强在**叙事与拿单**,本份强在**技术深度与工程交底**。两份配合,运营域就既会"讲"又会"接"。
> **素材来源**:某内河智能游艇示范项目**驾控台及智控系统技术协议 V1.00**(19页,合同附件级)+ 配套外装安装示意图、电气配盘接线说明、信号级接线定义表(xlsx)。乙方为智控/驾控台集成方,甲方为设计院/船厂链条;标的为 1 船双台(驾驶室主台 + 飞桥副台)、吊舱推进(2×40kW)、电池 357kWh、DC24V+AC220V。
> **怎么用**:这是**运营域的"技术协议对标卡 + 工程自检表"**。后续这个领域接到/起草任何驾控台·智控·智能航行的技术方案或技术应答,拿本卡逐条比对,查漏补缺、判断"专业不专业、完整不完整、安全合不合规"。每条都可反向当**需求挖掘清单**。

---

## 一、皇冠资产:12 把"懂行"评判尺子(可直接复用的领域自检表)

> 判断任何"船舶智控/智能驾控方案"是否真懂这行的探针——**能把"船"换成"车"而文字不变,就是不及格**。每条左侧打勾自检,右侧是一句话行规。

| # | ☑ | 评判维度 | 一句话行规 |
|---|---|---|---|
| 1 | ☐ | **安全链硬线优先** | 急停/安全停不经任何智能或软件,端子排硬接线直达推进控制箱;软件死了船也能停 |
| 2 | ☐ | **双电源主备 + 失电监测** | DC24V + AC220V 双线绝缘并行,中间继电器监测主/备电失电并上屏报警 |
| 3 | ☐ | **控制权切换带安全前置** | 智控↔有人、主台↔副台 切换前强制条件(0 转速 / 吊舱垂直归位 / 手柄回空挡)缺一不可 |
| 4 | ☐ | **推进无关的控制平台** | 同一智控栈兼容吊舱(0-330°回转·CAN)与螺旋桨(4-20mA),两套切换逻辑分别成立 |
| 5 | ☐ | **控制网与数据网物理分离** | CAN 走控制(手柄/方向机/雷达/推进),以太网走数据(视频/声呐/电池/4G),串口走报警 |
| 6 | ☐ | **船级社合规是底座非装饰** | 设备/材料满足 CCS 最新规范与修改通函,防护等级、试验、验收按 CCS 走,认可图会签 |
| 7 | ☐ | **环境/振动/盐雾按船级定档** | 工作温域、横纵倾、湿度、振动冲击、盐雾按真实航区定级,不是室温货架件 |
| 8 | ☐ | **信号级接线交底** | 每路信号标清:经/不经 VCU、无源/有源、模拟量/开关量/CAN/RS485/RS232,谁供线、多长 |
| 9 | ☐ | **供料责任划到根线** | 无疆供 / 船厂供 / 甲供 三方界面写死,标配长度写明,防交付扯皮 |
| 10 | ☐ | **冗余与应急接管** | 应急舵(偏航/转舵失效人工接管)、独立应急电源、多急停串联(任一路停整体) |
| 11 | ☐ | **船岸协同与数据上云** | 4G/5G 路由上云,岸端可取船端数据;高存储满足 N 天数据留存(本案≥30天) |
| 12 | ☐ | **"最低要求 + 认可图为准"双锚** | 技术协议只写最低要求,余量与合规由乙方兜底;最终尺寸/排布以甲方认可图为准 |

> 心法:这 12 把尺子里,**1/2/3/10 是安全四件套**(行业评审一票否决项),**4/5/8 是集成专业度**,**6/7/9/11/12 是合规与契约成熟度**。一份方案在安全四件套上含糊,后面写得再漂亮也过不了船检。

## 二、一份专业技术协议的标准骨架(8 段,缺段即不完整)

> 直接对标本案 19 页协议的章节顺序,起草/评审新方案时逐项打勾。

- [ ] **1 产品技术要求**:概述 · 供货范围(分类设备清单+数量)· 规范要求与标准 · 环境条件 · 电制 · 一般要求(颜色标记/语言/计量/HMI/电缆/EMC)
- [ ] **2 系统功能与技术指标**:驾控台结构 + 各子系统**逐个给型号 + 参数表 + 功能条目**(本案 12 个子系统,见第四节)
- [ ] **3 设计与制造控制要求**:设计人员资格 · 文件图样会签 · 制造特种工艺资格 · 防护包装运输贮存
- [ ] **4 试验和验收要求**:出厂试验项目(外观/电气性能…)· 验收(乙方试验+甲方复验)
- [ ] **5 产品包装与交付**:包装标准(GB/T 13384)· 交付方式与地点
- [ ] **6 到货期**:以商务合同为准 · 调试时间 · 多方清点签字
- [ ] **7 质量保证期限**:以商务合同为准
- [ ] **8 其他**:免费技术指导/培训 · 参加系泊/航行试验 · 报验给船厂/船检/船东 · 商务条款冲突以合同为准

> 经验:**第 2 章是含金量所在,占七成篇幅**;每个子系统都要"型号 + 参数表(电压/功耗/工作温度/防护等级/振动/盐雾/尺寸重量)+ 功能 bullet",不能只写功能不给型号参数。

## 三、智控系统能力清单:12 个子系统(参考配置基线)

> 一份完整的智能驾控方案应覆盖以下功能域;括号内为本案基线型号/指标,可作选型参考。

- [ ] **① 动力与转向控制**:按键/电源采集单元 + 协议转换盒 + **VCU(中枢)**;吊舱回转手柄或 4-20mA 手柄 + 电子方向机
- [ ] **② 电池监测**:工业交换机汇聚左右电池簇/直流板(Modbus TCP 上屏)
- [ ] **③ 报警与数据采集通信**:采集单元 + 串口解析(舱底水/液位/风机/通用·火灾·七氟丙烷报警 RS485)
- [ ] **④ 综合显示与报警**:15.6" 综合显控屏(1920×1080,4LAN/3CAN/2串口/1音频,≥30天存储)+ 千兆交换机
- [ ] **⑤ 导航集成(惯导)**:控制器 + 双天线(双天线定向 0.2°、姿态 0.1°、RTK 覆盖 50km、天线间距≥1.2m)
- [ ] **⑥ 声呐**:动态声呐(135° 成像、60m 深;通电前须浸水散热)
- [ ] **⑦ 毫米波雷达**:低速避障(40m 内、水平 120°、IP67、全天候)
- [ ] **⑧ 视频监控**:NVR + 4 路广角 + 电池仓摄像头 + **16 路智能边缘盒**(360°、深度学习、绊线/入侵/徘徊)
- [ ] **⑨ 音频**:功放 + 语音喇叭(实时播报,喇叭可扩展)
- [ ] **⑩ 4G/5G 云端通信**:路由器(船岸数据交互)
- [ ] **⑪ 急停**:M22 两常闭 + 保护盖,多急停串联
- [ ] **⑫ 综合显控界面**:电子界面 + 物理按键**双路控制机制**,预留综合显控接口

> 对外功能集(必须能讲清):**指点航行 · 航线/轨迹自主航行 · 紧急避停 · 定航速/定向 · 障碍物语音预警 · 智控/有人权限切换 · 主副驾驶权限切换 · 360° 视频**。

## 四、安全红线(一票否决项,直接复用)

> 这四条任一含糊,船检/船东评审会卡死。是本领域的"AI 红线"对位物。

1. **急停 = 纯硬线、绕开智能**:多急停**串联**,任一路触发**整体推进急停**;用**两对常闭触点隔离 1#/2# 推进**;给出 2 路/3 路串联接线图。**急停信号不经 VCU**。
2. **模式切换强制前置条件**:切智控前——吊舱方案需 0 转速 + 吊舱自动垂直归位;螺旋桨方案需手柄与给定转速为 0。**切回人工不要求 0 转速**。
3. **权限切换强制前置条件**:主↔副台切换前**两台手柄必须回空挡**;权限仲裁走 权限旋钮→VCU→CAN 内部判断(本案暂不上 PLC,预留升级)。
4. **冗余与人工接管**:**应急舵**(偏航/转舵失效时人随时接管)+ **独立应急电源** + 双电源主备失电监测。

## 五、接口与分工矩阵(防扯皮契约,直接复用模板)

> 本案最值得抄的工程习惯:把"哪根线谁供、多长、什么信号"写进合同级文档。新项目照填。

| 供方 | 典型负责项 | 标配/约定 |
|---|---|---|
| **集成方(无疆位)** | 智控全部盒子 · 雷达线束 · 声呐线束 · 惯导馈线×2 · 驾控台内部 CAN 网线 | 雷达 4m / 声呐 10m / 馈线 10m(标配,按实测加长) |
| **船厂** | 摄像头网线(6类 RJ45,≥5类优质)· 推进+直流板 CAN 定制网线 · 电池网线 · 扩展音频线(双股金银 0.95mm²) | 按长度提前铺设 |
| **甲供** | VCU↔1#/2# 推进控制箱 · VCU↔1#/2# 推进配电箱 的两路 CAN | — |

**信号级接线交底四要素**(逐路必标,源自接线定义表):
- [ ] 是否**经过 VCU**(推进闭环类经 VCU 走 CAN;安全/采集类不经 VCU)
- [ ] **无源 / 有源**输出输入
- [ ] **信号类型**:开关量 / 模拟量 / CAN / RS485 / RS232 / 以太网
- [ ] **A/B 两线成对** + **去向设备** + **供线方**

## 六、合规与环境基线(选型与应答的硬约束)

- [ ] **规范**:CCS《船舶应用电池动力规范》(2023)· 海事局《内河小型船舶检验技术规则》(2024)· CCS《材料与焊接规范》(2023)· GB/CB 标准 · 交船前生效的公约规范修正案
- [ ] **EMC**:GB/T 10250-2007,参考 IEC 60533 / IEC 60945
- [ ] **环境**:海水 -2~+32℃ · 环境 -25~+45℃ · 湿度 ≤95% · 横倾 ±15°/纵倾 ±7.5° · 水流 4.5m/s · 振动冲击
- [ ] **电制**:DC24V 双线绝缘 + AC220V/50Hz 单相双线绝缘
- [ ] **电缆**:低烟无卤、最小 1.5mm²、芯线长期工作温度 ≤85℃、通讯/模拟量用双绞屏蔽
- [ ] **设备等级**:控制盒 IP67 / 显控屏 IP66 / 惯导 IP68;振动 GJB150.16A、盐雾 GJB150.11(按航区定档)
- [ ] **细节**:316 不锈钢/铝制铭牌(黑底白字)· 主电路色标(交流绿黄棕/直流红蓝、地黄绿)· 全中文 HMI 与随机手册 · 控制箱内置原理图

> **一个必须主动接的张力点**:本案是**内河湖泊示范船**,设备却普遍按 **IP67 + 军标振动盐雾 + "50kn 高速艇/4 级海况/2.5m 浪高"** 选型——是**用海工/军工货架平台打内河项目**。正面=可靠性溢价 + 一套硬件通吃运营域(可平台化推广);代价=成本偏高,是采购砍价的切入点。方案里要用"平台化降本 + 可推广"正面对冲,别回避。

## 七、待澄清 / 开口项清单(交付物模板)

> 技术协议里凡标"**待定 / 若有 / 预留**"的,都要列进一张澄清表,挂责任人和回复时限(对应招式:开放项闭环)。本案的典型开口项:

- [ ] **供货数量口径**:协议"3 套" vs BOM"1 船双台"——是 N 条船还是含备件?
- [ ] **总用泵、舵角传感器**:标"若有/预留",本期是否纳入?
- [ ] **急停路数**:给了 2 路/3 路两版,本船扩展几路?
- [ ] **线束/馈线长度**:全部"按长度船厂提前铺设",需现场实测确认标配是否够
- [ ] **声呐安装支架与位置**:"根据实际需求沟通确认"
- [ ] **手柄/电子方向机是否独立供电**:配盘说明里两处"待确认"

## 八、检查用法(怎么拿这张卡对标一份新方案)

1. **先过安全四件套(第四节)**:任一含糊 → 直接判不合格,打回。
2. **用 12 把尺子(第一节)扫一遍**:能换成"汽车厂/通用 IoT"而不违和的条目 = 没懂这行,标红。
3. **对骨架(第二节)查缺章**:8 段缺哪段补哪段;第 2 章每个子系统是否"型号+参数表+功能"三件齐全。
4. **对能力清单(第三节)查缺子系统**:12 个功能域漏没漏;对外功能集能否讲清。
5. **对分工矩阵(第五节)查界面**:每根线是否落到了无疆/船厂/甲供,标配长度写没写。
6. **对合规基线(第六节)查硬约束**:CCS/EMC/环境/电缆/防护等级/铭牌色标。
7. **产出一张澄清表(第七节)**:把所有"待定/若有/预留"挂上责任人和时限。

> 三句话总结这份标杆教给运营域的事:**安全不交给软件、界面写到根线、合规是底座不是装饰**。
