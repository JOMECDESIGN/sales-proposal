# 技术方案素材 · 船舶态势感知与远程控制(自有方案)

> **业务领域**:船舶智能化 / 智能航运(元启晨星 **运营域** · 感知—控制—远控维度)。
> **素材来源**:自有方案《船舶态势感知与远程控制落地技术方案》(以 YOLO 视觉为核心,融合雷达/LiDAR/AIS/GNSS-RTK/FOG-IMU/气象/海流/卫通与岸基远控)。
> **怎么用**:这是**核心产品"控"那一维的底稿**——[核心产品说明·第六节 控制与 AI 责任边界](船舶-智能船机交互系统-核心产品说明.md) 的分级控制闭环即出自此。写涉控/远控/自动驾驶方案、谈感知融合与安全降级时,从这里取弹药(对应 🛠️ Sales Engineer、📐 UX Architect)。
> **关键区分**:本方案是**自有控制**(自研分级闭环);[岸基远控综合平台素材](船舶-岸基远控综合平台-中远海运重工舟山-项目素材.md) 里"不替代第三方船控"是**那个招标项目的特定约束**,两者别混用。

## 一、一句话定位

> 不是"单一感知大屏",而是一条**完整闭环**:多源采集 → 时间同步 → 坐标统一 → 目标级/场景级融合 → 风险评分 → **建议/辅助/自动/远控**控制输出 → 回放与审计。

对标主流厂商方向(ABB Marine Pilot Vision 多传感器融合+虚拟船模;Wärtsilä S.P.E.C.S 360°盲区消除+AR;Avikus HiNAS 视觉+AIS+雷达+热像+自动航线/速度+岸基云;Kongsberg/DNV 远程中心+自动靠离泊+可认证自动化),路线一致且**可认证、可演进**。

## 二、七层设备链路(采集→融合→决策→执行→回传)

| 层 | 设备 | 作用 |
|---|---|---|
| 环境与目标感知 | 相机/热像/LiDAR/雷达/AIS/气象/波流 | 识别目标、远距协作目标、近场厘米测距、环境量入风险模型 |
| 船体态势 | GNSS/IMU/计程仪/测深 | 本船位姿速度+水深;为所有外部感知做时空配准 |
| 融合与 AI | 边缘算力/融合处理器/避碰计算机 | **核心价值层**:时间同步、坐标变换、多目标跟踪、风险归因、决策出参 |
| 交互与呈现 | AR/SVM/报警矩阵 | HMI、告警、人机确认 |
| 远程传输 | VSAT/Starlink/4G/5G/网关 | 岸基近实时孪生与控制通道 |
| 岸基 RCC | 虚拟驾驶台/服务器集群 | 多船监视、接管、事件复盘 |
| 执行层 | 遥控接口/伺服阀/自动舵/主机/侧推 | 动作与反馈,**必经权限仲裁+安全包线+故障切换** |

**必补传感器(否则只"看见船外"、形不成全船安全态势)**:测深仪(触礁/UKC)、舱底高液位(漏水)、机舱火警感烟感温(火灾,SOLAS II-2)、门禁/禁区/舱口、UWB/BLE 人员定位、机舱压力/阀位/泵/电气、抗干扰 PNT 备份(GNSS 欺骗)。

## 三、九类风险场景(前三类点名,后六类常见扩展)

碰撞/障碍 · 恶劣天气 · 触礁/浅滩 · 人员落水 · 火灾/烟雾 · 漏水/进水 · PPE 违规/禁区闯入 · 通信中断/网络攻击 · GNSS 欺骗/失锁。(靠离泊=碰撞在港区 ODD 的专用工况,由 SVM+LiDAR+侧推+DP/自动舵强化。)

每个场景压成"采样—同步—融合—动作" profile(示例):
- **碰撞**:YOLO/RTMDet 检测 + ByteTrack/OC-SORT 跟踪 + 雷达/AIS 关联 + CPA/TCPA + COLREG 分类 + VO/MPC/RRT* → 预警/AR/改向减速/自动避碰/RCC 接管。
- **触礁**:ENC/DEM 匹配 + UKC 估计(=海图深度+潮位−吃水−下蹲−涌浪余量−不确定度)+ bathymetry-aware RRT* → 改线/减速/禁越界自动转向。
- **落水**:目标检测升级为**事件检测**(电子围栏+姿态/轨迹落海判别+热像复核+UWB 失联)→ MOB 告警/自动打点/回放/SAR 通知。

## 四、三层融合 + 双总线 + 时空基准(成败关键)

- **三层融合**:数据级(相机-LiDAR BEVFusion/点云/雷达栅格)· 目标级(JPDA/EKF/UKF/IMM/PHD)· 场景级(规则引擎+图优化/MPC/RRT*/VO)。
- **双总线(强烈建议)**:**ROS 2/DDS** 承载毫秒级传感器与控制(QoS 可调);**Kafka** 承载录制/回放/状态/事件(log compaction 保"最后可信状态");NMEA 2000/IEC 61162 对接传统航电。
- **时空基准**:GNSS 1PPS + PTP 主时钟(失锁切 IMU 守时);统一 ECEF→ENU→船体→传感器外参;每条消息带 `source_ts/ingest_ts/frame_id/seq/ptp_offset` 五字段,**支持事后审计与因果回放**。
- **落库**:高频遥测进 **TimescaleDB**(与底座同栈);视频/点云/事件包进对象存储;模型用 MLflow;指标进 Prometheus。

> **风险评分**:场景统一壳(可信度/数据缺失/告警抑制)+ 场景专用特征头(CPA/TCPA、UKC、风浪、火警…),输出统一 `ShipRiskEvent` JSON Schema(event_id/scene/severity/risk_score/confidence/ownship/targets/recommended_action)。

## 五、⭐ 分级控制闭环 = 责任边界(核心产品同源)

控制权按 **风险评分 + 置信度 + ODD + 关键传感器健康** gating,分四档:

| 模式 | 触发 | 输出 | 人确认 |
|---|---|---|---|
| 建议 advisory | 风险<0.55 或置信不足 | HMI 建议、报警提示 | 否 |
| 辅助 assist | 0.55–0.75 | AR/SVM 叠加、限制危险命令、建议侧推/减速 | 是 |
| 自动 auto | 0.75–0.90 且 ODD 内、传感器健康 | 避碰/减速/保持航道/靠泊微调 | 视场景 |
| 远控 takeover | ≥0.90 或复合异常、通信正常 | RCC 接管,船端保留安全壳 | RCC 值守 |

- **执行链三道闸**:权限仲裁 → 安全包线(限幅/危险命令拦截)→ 故障切换;远控额外需 cyber gateway 放行 + 通信健康 + 控制租约 + 命令签名/白名单。
- **Fail-safe 降级**:断网→本地自治禁新远控;传感器失效→切权重;GNSS 欺骗→切惯导禁高自动;不确定性高→降级 advisory only;RCC 异常→船端接管 fail-safe。
- **AI 对法定系统**:火警/进水等法定安全系统,AI 是**减少误报+区域定位+提高先验**的增强,**非替代**。

## 六、自动驾驶演进(IMO MASS,不一步到位)

决策支持(L1,全航段)→ 有监督自动驾驶(L2,港区/特定航线/低速)→ 远控接管与高等级自主(L3,重复航线/受限水域)。**最先可商业化的高价值 ODD = 重复航线自动靠泊**(SVM+LiDAR+码头地图+侧推)。可复用:COLREG-aware planner、Velocity Obstacle/MPC、Grounding-aware RRT*。

## 八、工程架构与硬件分档

- **软件**:容器化微服务 + 双总线(ROS2/Kafka)+ 双数据库(TimescaleDB/对象存储)+ 模型治理(MLflow)+ 监控(Prometheus)+ 流媒体(GStreamer/DeepStream)。
- **硬件分档**:轻量(单 Jetson AGX Orin + 备 Orin NX,小型沿海/验证船)· 标准(双 x86 边缘 + L4 或双 Orin + NVMe,一般商船/靠泊/远控预留)· 重载(双 x86 + L4/A2 + 独立导航单元,多路视频/LiDAR/远控/孪生并行)。
- **网络优先级**:控制命令与心跳 > 风险事件与低码流关键视频 > 融合目标流 > 全分辨率回传 > 历史批同步;卫通双链路冗余,近岸 4G/5G、远海卫通。

## 十一、素材速记(新人必知)

- **一句话**:多源融合 → 风险评分 → 建议/辅助/自动/远控 的完整闭环,不是感知大屏。
- **架构关键四件**:PTP 时钟 · ROS2+Kafka 双总线 · 统一坐标系 · 五字段时间戳(可回放可审计)。
- **三层融合**:数据级 / 目标级 / 场景级;**风险引擎** = 统一壳 + 场景特征头 + ShipRiskEvent Schema。
- **⭐ 控制责任边界**:四档(建议/辅助/自动/远控)按风险+置信+ODD+传感器健康 gating;三道闸(仲裁→安全包线→故障切换);永远向安全侧降级。**核心产品同源。**
- **演进**:IMO MASS L1→L2→L3;先做自动靠泊这个高价值 ODD。
- **同栈证据**:TimescaleDB 与 Yog IOT 底座一致;标准化、可认证、可演进。
- **标准锚点**:COLREG Rule 7、IMO MASS、SOLAS II-2、IHO S-100(S-101/102/104/111/124/129)、IACS UR E26/E27。
