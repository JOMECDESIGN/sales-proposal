# 检查清单 · 船舶方案送审对标自检

> **对应招式**:招 16 方案自检 / 答辩演练。
> **谁来填**:🏋️ Sales Coach 主持,🛠️ Sales Engineer × 📐 UX Architect × 🐑 Project Shepherd 逐项过。
> **用法**:方案定稿前,**站在总体所 / 船检专家视角**逐条打勾。每条只有"是 / 否 / 不适用"三态;凡打不出勾的,就是答辩会被问、被扣分的地方。配套素材见 [案例标杆/船舶-设备接入接口与协议覆盖-知识卡片.md](../案例标杆/船舶-设备接入接口与协议覆盖-知识卡片.md)。
> **怎么算过**:A、B、C 三组全绿,方可送审;D 组为加分项。

---

## A. 责任边界(最先被审 · 一票否决)

- [ ] 设备已按 **涉控 / 只读 / 法定取证安全** 分三堆,描述口径各不相同,没有笼统写"接入"。
- [ ] 涉控对象(自动舵 / 推进车令 / 联锁 PLC)写明:**辅助建议 → 三道闸 → 人确认**,执行权留在既有控制系统,命令下发**不在采集链路**、而经受控网关。
- [ ] 法定安全系统(VDR/S-VDR、火警、进水)写明**只读镜像 / 辅助**,不改写、不替代,只降误报 / 辅助定位。
- [ ] 自动化分级(建议 / 辅助 / 自动 / 远控)有 **fail-safe** 与"人始终可接管"的明文,且与 IMO MASS 演进对齐。
- [ ] 风险评分 / AI 建议明确为**参考输入**,是否动作由规则 + 安全包线 + 人共同决定,不由 AI 单独决定;全程可回放、可审计。

## B. 接口与协议(覆盖面 + 准确性)

- [ ] 协议覆盖按 **4 大类**组织:航海数据(NMEA 0183/2000·IEC 61162)/ 工业总线(Modbus·CAN J1939·Profinet·EtherNet·IP·OPC UA)/ 视频(RTSP·ONVIF·SDI·HDMI)/ 硬接线 I/O(AI·AO / DI·DO / RS-422·485)。
- [ ] 标准号**显名**:NMEA 2000 = IEC 61162-3、网络型 61162-450;雷达 IEC 62388、ECDIS IEC 61174、显示 IEC 62288。
- [ ] **12 类典型设备全部进附录 D**(I/O list),不是只列几行示例;附录 D 比正文全、不矛盾。
- [ ] 附录 D 五列闭环:**设备 — 信号点 — 协议/接口 — 量纲 — 隔离方式(只读/旁路/隔离/受控)**,量纲与点性不缺。
- [ ] **§协议表 / 附录 B 对外接口概览 / 附录 D 三处同源**,交叉对照不自洽处已消除;完整 ICD 注明随详细设计提供。
- [ ] 采集侧统一**旁路、隔离、只读**接入,不加载既有总线,有明文。

## C. 术语与计量(船检逐字看)

- [ ] 定位写 **GNSS**(非 GPS),并体现多星座 + **北斗 BDS**。
- [ ] **计程仪 ≠ 测深仪**:航速(kn,对水/对地)与水深(m)分行,量纲正确。
- [ ] 罗经**分型**(磁 / 陀螺 / 卫星);航向 HDT/THS 与艏摇率 ROT 区分。
- [ ] 摄像头写 **CCTV / 视频监控**,注明非法定、仅辅助值守,不与法定瞭望混淆。
- [ ] 全文出现的协议 / 标准缩略语,**附录 E 缩略语表均可查到**(无"正文有、表里无")。

## D. 规范符合性与送审完整性(加分项)

- [ ] 规范矩阵覆盖:CCS《智能船舶规范》、IEC 60945 / 60533、IACS UR E10 / E26 / E27、SOLAS V/22(视野)、IMO MSC/Circ.982(人因)、IEC 62288 / 61174 / 62388,每行有"对应措施 + 验证方式"。
- [ ] 驾驶台 / 智慧驾驶舱写明:满足 SOLAS V/22 视野、按人因导则布置、不替代法定 ECDIS 与雷达(仅辅助综合呈现)。
- [ ] **CCS 为路线而非现状**:实船应用 → 稳定运行后再推进认证,无"已认证"措辞。
- [ ] 配置 / 数值类一律标 **示例 / 待定**,留详设填实;附录 C(BOM + 供电功耗)结构在位。
- [ ] **去 AI 腔**:无沟通痕迹标注,无"融合诊断 / 智能识别 / 精准"等过度措辞;AI 描述已弱化为"辅助"。

---

> **打勾自检**:
> - [ ] A 组全绿(责任边界是答辩的命门,任一缺失即不送审)。
> - [ ] B、C 组全绿(接口覆盖、术语计量经得起逐字看)。
> - [ ] 凡打"否"的条目,已在《待定项与风险登记》登记并指派责任人。
> - [ ] 答辩前用本清单做过一次**模拟提问**(每条"否"预设一个专家追问 + 回答口径)。
