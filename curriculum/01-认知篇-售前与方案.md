# 认知篇 · 售前与方案

> **写给谁**:完全没接触过售前的应届/转岗新人。
> **目标**:读完这一章,你能说清三件事——售前到底是干什么的、方案为什么是"销售武器"、以及你该有的几个思维转变。
> **这是整套训练营的第一课,不教招式,只换脑子。**

---

## 一、售前是什么:把"能做"翻译成"该买"

一句话:**售前(Pre-Sales)站在销售和技术中间,负责让客户相信"你们能解决我的问题,而且值这个钱"。**

- 销售管"关系和成交";
- 交付管"做出来";
- **售前管中间最难的一段——把技术能力,变成客户愿意买单的理由。**

新人常见误解 vs 真相:

| 误解 | 真相 |
|---|---|
| 售前 = 写技术文档 | 售前 = 用技术证据做**说服** |
| 把产品讲清楚就行 | 把"客户的问题怎么被解决"讲清楚才行 |
| 参数越多越显专业 | 客户记不住参数,只记住"这对我有什么用" |
| 方案是给技术评委看的 | 方案要同时喂饱老板、评委、采购三种人 |

---

## 二、方案是什么:不是说明书,是销售武器

很多新人把方案当"产品说明书"来写——这是头号误区。

> **方案 = 一份说服文档。** 它的唯一目的,是让客户做出"选我们"的决定。

记住这条判断标准(贯穿整套教材):

> **如果把客户名字换成另一家公司,你的方案照样成立——那它已经在输了。**

好方案和差方案的分水岭:

| 差方案(合规清单) | 好方案(说服文档) |
|---|---|
| 罗列"我们有什么功能" | 回指"你那个麻烦,我们怎么解决"(招1) |
| 堆参数 | 把参数翻译成业务价值(招2) |
| 一个语气写到底 | 给老板/评委/采购各一条动线(招3) |
| 通篇"领先""强大" | 每个主张都配证据(招6) |
| 结尾"期待合作" | 给一个具体的下一步(招15) |

> 括号里的"招N"就是后面 `02-方法论-招式卡片.md` 要学的。这一章只要先建立"方案是用来说服的"这个认知。

## 三、客户为什么买:决策链上的三种人

你写的每一份方案,背后至少有三种人在看,他们的问题完全不同:

| 角色 | 他真正在问 | 你要给他什么 |
|---|---|---|
| **决策者**(老板/分管领导) | 为什么是现在?值不值?有没有风险? | 价值、信任锚点、低风险的下一步 |
| **技术评委** | 你们交得了货吗?技术可行吗? | 架构、证据、案例 |
| **采购** | 合不合规?性价比如何? | 验收标准、对比表、清晰报价 |

> 这就是为什么"一份平均主义的方案谁都喂不饱"。售前的功夫,一半在于**同时照顾这三种人**。

---

## 四、新人必须完成的四个思维转变

1. **从"功能视角"到"痛点视角"**
   别问"我们的产品有什么",先问"客户的麻烦是什么"。能力是用来补痛点的,不是用来炫的。

2. **从"讲全"到"讲准"**
   把所有功能都写进去 ≠ 专业。抓住客户最痛的那几点讲透,胜过面面俱到。

3. **从"我觉得好"到"有证据"**
   "强大""领先""丰富经验"在评委眼里是噪音。每句主张后面要么有数字、案例、方法名,要么删掉。

4. **从"写完就完"到"推动成交"**
   方案的终点不是"交出去",而是"客户迈出下一步"。结尾没有具体行动,等于白写。

## 五、在这套体系里,你的真正本事是什么

本仓库有一支 7 人 Agent 团队(售前工程师、方案策略师、项目护航者……),它们能替你执行 16 招里的大部分动作。

所以,**纯新人最该练的,不是把招式背熟,而是这三件更值钱的事**:

1. **判断**:当前这一步该用哪招、该叫哪个 Agent。
2. **喂料**:给 Agent 足够、准确的输入(尤其是客户原话和真实痛点)。
3. **验收**:看得懂 Agent 的产出好不好,能挑出空话和没证据的主张。

> 这三件事,正是 `03-Agent团队使用手册` 和 `04-流程篇` 要带你练的。
