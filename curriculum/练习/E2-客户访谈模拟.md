# 练习 E2 · 客户访谈模拟

> **练什么**:招 1 痛点回指、招 3 受众分层、招 12 痛点收敛。学会"问出痛点",而不是"被动接需求"。
> **配合谁**:🛠️ Sales Engineer(陪练痛点收敛 + 干系人地图/受众分层)+ 🏹 Proposal Strategist(陪练受众阅读动线)。
> **耗时**:45-60 分钟。

---

## 场景设定(残缺的客户背景)

> 你接到一条线索,信息很不完整——这就是真实售前的常态。
>
> **客户**:某区域航运公司,有 30 条内河货船。
> **已知**:负责人在一次饭局上抱怨"船上设备一堆牌子,数据各管各的""出了事故调监控特别费劲""新招的船员上手太慢"。
> **未知**:谁拍板?预算多少?有没有合规deadline?现有系统是什么?

新人常见错误:拿着这点信息直接开写方案。**正确动作:先设计访谈,问出缺口。**

---

## 任务一:设计访谈提纲

针对三类受众各设计 3 个问题。目标是问出痛点的**根因**和**决策链**,不是问"你想要什么功能"。

| 受众 | 你要问出什么 | 你的 3 个问题 |
|---|---|---|
| 决策者(分管领导) | 为什么是现在?预算与决策权 | |
| 业务负责人(船队管理) | 真实痛点与日常麻烦 | |
| 技术/信息负责人 | 现状系统、数据现状、合规要求 | |

> 提示:好问题是开放式的、追问"为什么"和"现在怎么应对",而不是"你要不要 X 功能"。

---

## 任务二:痛点收敛

把饭局上的三句抱怨,套用招 12,收敛成"表象→根因→后果":

- **表象**(客户能直接感受的):
- **根本原因**(归到一个底层成因):
- **后果**(不解决会怎样):

> 对照船舶标杆 P08 的范例:表象(体验差/管控弱/船岸割裂)→ 根因(底层数据割裂)→ 后果(无法复制、无法积累资产)。

---

## 任务三:用 Agent 演练

让 🛠️ Sales Engineer **扮演这位客户负责人**,你用任务一的提纲做一轮模拟访谈:
> "请你扮演这家航运公司的业务负责人,带点防备、不会主动说全。我来访谈,你按真实客户的反应回答。结束后,点评我哪些问题问到了点子上、哪些是无效问题。"

**过关线**:
- [ ] 问出了至少一个"未知"项(决策权/预算/合规deadline/现状系统)。
- [ ] 痛点收敛到了一个根因。
- [ ] 没有出现"你要不要 XX 功能"这种诱导式问题。
