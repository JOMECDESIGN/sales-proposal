## 实战剧本(对照流程篇五段)

### 第①段 · 挖需求 → 叫 🛠️ SE + 🗺️ Account
- [ ] 产出痛点清单(表象→根因→后果)
- [ ] 产出三类受众分层表
- 模板:`模板库/`(暂用痛点表)

### 第②段 · 定方向 → 叫 🏹 Strategist(+🧭 PM)
- [ ] 3-5 个赢主题(用 Strategist 已有的赢主题矩阵模板)
- [ ] 6 章决策问题弧(招 16)
- [ ] 一页执行摘要
- [ ] 选定最强信任锚点,前置(招 11)

### 第③段 · 搭主体 → 叫 🛠️ SE + 📐 UX + 🧭 PM
- [ ] 架构主线一句口诀(招 7)
- [ ] 技术章节:每条能力回指痛点 + 翻译价值
- [ ] 价值对比矩阵(招 8)→ 模板:`模板库/升级对比矩阵.md`
- [ ] 案例:挑战→核心技术→成果

### 第④段 · 出计划 → 叫 🐑 Shepherd
- [ ] 交付物与验收表 → 模板:`模板库/交付物与验收表.md`
- [ ] 关键路径日历 → 模板:`模板库/关键路径日历.md`
- [ ] 待定项与风险登记 → 模板:`模板库/待定项与风险登记.md`
- [ ] 三步走合作路径 + 退出机制(招 14)

### 第⑤段 · 拿单 → 叫 🏋️ Coach + 🗺️ Account
- [ ] Coach 全盘自检,逐条改
- [ ] 临门一脚 CTA(招 15):低风险下一步 + 时限承诺
- [ ] 帮客户向上汇报的素材(政策/预算)

| # | 验收项 | 得分 |

| 1 | 痛点都回指了客户原话/事实 | |
| 2 | 痛点收敛到一个根因 | |
| 3 | 赢主题换客户名就不成立(够具体) | |

| 5 | 有可复述的记忆锚点 | |
| 6 | 最强证据前置 | |

| 8 | 架构有外行也懂的主线 | |
