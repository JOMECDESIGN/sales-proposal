# 05-排版篇 · 把方案变好看(开源 Markdown 出片产线)

> **为什么有这一篇**:对元启晟来说,**版面效果是信任的一部分**——同样的内容,排版决定客户第一眼把你归到"靠谱供应商"还是"草台班子"。
> 本仓库现在产出的是 **纯 `.md`**(招式、模板、Agent 产出都是 Markdown)。这一篇解决最后一公里:**怎么把 `.md` 一键变成"打得出去"的 PDF 文档 / 路演 deck**,而且**不改变现有写作流程**——你照旧写 Markdown,工具负责好看。
> **谁主管这件事**:📐 UX Architect(可视化职责的延伸)。详见文末「与 Agent 团队的衔接」。

---

## 一、先分清:售前方案有两种"版面"

| 形态 | 用途 | 客户怎么拿到 | 对应工具路线 |
|---|---|---|---|
| **A. 正式方案文档** | 投标、留档、随合同附件 | PDF / Word,可打印、可盖章 | 文档路线(Pandoc / Quarto) |
| **B. 答辩 / 路演 deck** | 现场讲、评审会、Demo Day | 投屏 PPT / 网页幻灯 | 幻灯路线(Marp / Slidev) |

> 两大主营各有侧重:
> - **船舶智能化(元启晨星)**= 偏 **A 正式文档**(投标、政策申报、留档)→ 主推 **Pandoc + Eisvogel / Quarto**。
> - **智能座舱 / 展陈**= 偏 **B 现场演示**(Demo Day、评审答辩)→ 主推 **Marp / Slidev**。
>
> 同一份方案,内容写一次(Markdown),A、B 两种版面**各导一次**即可——这正是 Markdown 出片产线的价值。

## 三、推荐默认(给元启晟的两套组合拳)

> 选型原则:**学习成本最低、可复制可量产、中文不踩坑、能进 CI 自动出片。**

- **答辩 / Demo → Marp**:写法就是带 frontmatter 的 Markdown,一条命令出 PPTX/PDF,跟本仓库现有 `.md` 资产**无缝衔接**。要更炫再上 **Slidev**。
- **正式投标 → Pandoc + Eisvogel**(或 Quarto):把船舶方案这类长文档,排成有封面/页眉/页脚的专业 PDF。

## 四、怎么植入本产线(分步,可照抄)

### 步骤 0 · 约定目录(一次性)

在仓库根加这几个目录(放渲染产物与主题,**与教材源文件分离**):

```
sales-proposal/
├── publish/            # 出片配置区(新增)
│   ├── themes/         #   存放 Marp/自定义 CSS 主题、Eisvogel 模板
│   ├── slides/         #   答辩 deck 的 .md 源
│   └── docs/           #   正式文档的 .md 源
└── build/              # 渲染产物 PDF/PPTX(建议 .gitignore,不入库)
```

> 现有 `curriculum/模板库/*.md` 内容**不用动**:出片时把要交付的章节拷进 `publish/`,或直接对模板文件跑命令即可。

### 步骤 1 · 幻灯:Marp(零安装试一次)

给任意 `.md` 顶部加 frontmatter,即可被 Marp 识别为 deck:

```markdown
---
marp: true
theme: gaia          # 商务首选;或换 marpstyle 的自定义主题
paginate: true       # 页码
backgroundColor: #fff
---

# 元启晨星 · 船舶智能化方案
#### 致 XX 航运 · 2026

---

## 痛点 → 价值(一页一观点)

- 现状:**人工巡检**,漏检率高
- 我们:AI 视觉巡检,**效率 +50%**
```

一条命令出三种格式(`npx` 无需预装):

```bash
# PPTX(给评审现场用)
npx @marp-team/marp-cli@latest publish/slides/deck.md --pptx -o build/deck.pptx
# PDF(留档/发客户)
npx @marp-team/marp-cli@latest publish/slides/deck.md --pdf  -o build/deck.pdf
# 用自定义主题
npx @marp-team/marp-cli@latest publish/slides/deck.md --theme publish/themes/yuqi.css -o build/deck.pdf
```

> VS Code 装官方扩展 **Marp for VS Code**,可边写边实时预览,所见即所得。

### 步骤 2 · 正式文档:Pandoc + Eisvogel

```bash
# 1) 装 Pandoc + TeX(xelatex 才能排中文);Eisvogel 模板放到 publish/themes/
# 2) 一条命令把长方案排成专业 PDF:
pandoc publish/docs/船舶方案.md \
  -o build/船舶方案.pdf \
  --template publish/themes/eisvogel.latex \
  --pdf-engine=xelatex \
  -V CJKmainfont="Noto Serif CJK SC" \   # 中文字体,思源宋体/黑体均可
  -V mainfont="Noto Serif CJK SC" \
  --toc                                  # 自动目录
```

文档顶部用 frontmatter 控制封面信息:

```markdown
---
title: "元启晨星 · 船舶智能化整体方案"
subtitle: "致 XX 航运集团"
author: "元启晟 售前团队"
date: "2026-06"
titlepage: true        # Eisvogel 封面页
toc-own-page: true
---
```

> **中文唯一的坑**:必须用 `--pdf-engine=xelatex` 且指定 `CJKmainfont`,否则中文显示为空白方框。字体推荐开源的 **思源宋体 / 思源黑体(Noto CJK)**,免费可商用。

### 步骤 3 · 自动出片(可选,进 GitHub Actions)

让每次 push 自动生成最新 PDF/PPTX,产物挂到 Actions artifact:

```yaml
# .github/workflows/build-deck.yml
name: Build proposal artifacts
on:
  push:
    paths: ["publish/**"]
  workflow_dispatch:
jobs:
  marp:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Render slides
        run: |
          npx @marp-team/marp-cli@latest publish/slides/deck.md --pdf  -o deck.pdf
          npx @marp-team/marp-cli@latest publish/slides/deck.md --pptx -o deck.pptx
      - uses: actions/upload-artifact@v4
        with:
          name: proposal-deck
          path: |
            deck.pdf
            deck.pptx
```

> 正式文档(Pandoc/xelatex)同理,用 `pandoc/latex` 官方 Docker 镜像跑一步即可。

## 五、与 Agent 团队 / 招式的衔接

| 环节 | 主导 Agent | 新增动作 |
|---|---|---|
| 内容定稿(Markdown) | 各 Agent 按 `04 流程篇` 产出 | 不变 |
| **版面渲染 / 出片** | 📐 **UX Architect** | 选路线(A 文档 / B 幻灯)→ 套主题 → 跑命令 → 自检版面 |
| 答辩前彩排 | 🏋️ Sales Coach | 拿**渲染后的 deck**(不是裸 md)演练,检查投屏效果 |

**版面自检清单(出片前过一遍):**

- [ ] 一页一观点,文字不超过 6 行(deck);正式文档有目录、页码、封面。
- [ ] 关键数字/对比用**表格或图**,不堆段落。
- [ ] 中文字体正常显示(无方框),中英文混排不挤。
- [ ] 配色克制:主色 + 中性灰,别超过 3 色。
- [ ] 导出后在**目标设备**(投影仪/打印)上实看一遍。

## 六、一张图记住怎么选

```
要现场讲?
 ├─ 要快、要能导 PPTX 给评审 → Marp(默认)
 └─ 要最炫、有动画/组件      → Slidev
要交正式文档?
 ├─ 要像招标书的专业 PDF     → Pandoc + Eisvogel(默认)
 └─ 一稿多出 PDF/Word/网页   → Quarto
要发公众号/网页版案例?       → markdown-nice
```
