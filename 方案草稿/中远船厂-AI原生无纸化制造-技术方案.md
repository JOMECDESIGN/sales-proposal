# 第一部分 · 立项依据与评判标尺

## 0. 文档目的、范围与读者

**目的**:为中远船厂(造船业务)建设一套 AI 原生的无纸化先进制造应用,提供技术论证、总体架构、核心能力设计与集成方案。本文先把"造船这门生意的特殊性"讲透,再据此推导技术选择,避免把通用离散制造 MES 的假设硬套到船厂。

**范围**:覆盖从三维设计模型流转 → 工艺/施工包 → 计划排产 → 一线移动施工 → AI 预审 → 人工复核 → 报验/见证 → 状态流转 → 全过程归档的现场制造执行域,以及其上的 AI 能力层与知识资产层。**不重写**船厂既有的 PDM/CAPP/MES/ERP/APS,而是定义本应用与它们的边界与接口。

**读者**:船厂生产/工艺/质量/信息部门技术评审、设计院所接口人、船级社/船东数字化接口人,以及承研方研发团队。

## 1. 造船行业评判标尺:12 把尺子

> 这一章是全文的"地基"。任何号称服务船厂的数字化/AI 方案,都应当先用这 12 条自检。一个不懂造船的方案,可以把"船厂"二字替换成"汽车厂/钢构厂"而文字不变——这就是不及格。下面每条给出**为什么重要**、**及格线**与**优秀线**,后续技术章节都会回扣这把尺子。

> 下面 1.1–1.12 逐条详解。每条用"❌ 不懂行的写法 vs ✅ 懂行的写法"作对照——这组对照本身就是检测一份方案专业度的探针:能写出 ✅ 那一列,才算真把这门生意看进去了。

### 1.1 以中间产品/分段为组织单元

- **机理(行规)**:现代造船自上世纪推行"成组技术 + 区域造船法/分道建造"以来,生产的基本对象不是"一个零件流过一条线",而是**中间产品**——零件→部件→小组立→中组立→大组立→分段→总段→搭载。整个工厂的计划、物料、质量、状态都挂在这棵"分段树"上。
- **❌ 不懂行**:对象模型是"工序 + 工位 + 工单",像汽车产线;BOM 是一张平表;状态只有"未开工/进行中/完成"。
- **✅ 懂行**:核心对象 = 船号→分段/总段→区域/托盘→工序→检验点的层级树;一个分段可在不同阶段被不同工种"接力",状态能按分段、按区域、按工序三个维度聚合与下钻。
- **对本方案**:全链路数据模型即分段树(第6章对象模型、第12章统一编码),否则后面所有 AI 与看板都建在错误地基上。

### 1.2 对接船台/船坞周期这个主瓶颈

- **机理**:船坞/船台是全厂最稀缺、最贵的资源,一条船的交期本质由"坞期"决定。坞内多占一天,就少造别的船,损失以日计的巨额机会成本。船厂厂长真正盯的 KPI 是**坞内一次合拢成功率**和**坞期压缩**。
- **❌ 不懂行**:通篇谈"流转效率""签字提速",从不出现"坞期/船台周期"这个词。
- **✅ 懂行**:把数字化动作显式接到坞期——搭载前的精度预检、缺漏件清零、报验前置,目的都是"让分段进坞即可装、一次装对"。
- **对本方案**:KPI 体系以坞内一次合拢成功率为首(第5章);长远规划里数字孪生的落点是搭载网络/坞期仿真(第17章),而非泛可视化。

### 1.3 覆盖三检制 + 报验 + 见证点

- **机理**:造船质量控制是多层级、且含**强制外部见证**的:自检→互检→专检(厂 QC)→**船东验船师→船级社验船师**。其中 Hold Point(停工待检,未验不得继续)和 Witness Point(见证点)是**合同与规范强制**,卡住整条生产链。
- **❌ 不懂行**:把"报验"简化成内部一道"质检员签字";完全没有船东/船级社的见证排期与停工待检概念。
- **✅ 懂行**:报验是一个含外部干系人的工作流——申请、排期、见证(可远程)、签认、退审、整改、复验,且区分 Hold/Witness Point。
- **对本方案**:第11章把船东/船级社在线见证单列为核心能力,而非一行表格。

### 1.4 壳舾涂一体化 + 托盘/区域管理

- **机理**:现代造船把船体(壳)、舾装(舾)、涂装(涂)在同一区域**一体化、并行、尽量前移**(预舾装、单元化舾装)。舾装物料按**区域 + 托盘(Pallet)**组织齐套,一个托盘 = 某区域某阶段要装的一整套料/件/管/电缆。
- **❌ 不懂行**:物料就是 ERP 里一张通用领料 BOM,"齐套"是个总数百分比;没有"托盘""区域""预舾装率"。
- **✅ 懂行**:齐套按托盘/区域算;舾装工单绑定托盘;区域内多专业(管/电/暖通/铁舾)有防碰撞与施工顺序约束。
- **对本方案**:第12章 ERP 接口按区域/托盘组织齐套;第9章舾装工单与托盘绑定、做多专业冲突提示。

### 1.5 精度管理(Accuracy Control)

- **机理**:钢板下料到分段合拢,误差会累积。船厂用**余量/补偿**和**精度造船(无余量造船)**来控制——下料留多少余量、何时割除、合拢口能否一次对中,直接决定坞内是否要现场修割(极费工费坞期)。现代做法靠**全站仪/三维激光扫描**实测驱动决策。
- **❌ 不懂行**:完全不提精度/余量,默认"按图做就能装上"。
- **✅ 懂行**:把测量数据(全站仪/扫描)纳入闭环,余量决策可追溯,搭载前做精度预检。
- **对本方案**:第13章接入精度测量数据;它是 1.2(坞期)的直接技术支撑。

### 1.6 质量可追溯的数据血缘链

- **机理**:船级社要求关键质量可**全程反查**——这条焊缝由哪个持证焊工焊的(焊工钢印/资质↔WPS/PQR)、做没做 NDT(RT/UT/MT/PT)、所用钢板的炉批号/材质单(钢印移植)。这是一条贯穿"人-工艺-检测-材料"的血缘链。
- **❌ 不懂行**:质量记录 = 拍张照 + 质检员打钩;焊工、WPS、NDT、材质各自孤立。
- **✅ 懂行**:任一焊缝/构件可一键反查焊工资质、对应 WPS、NDT 报告、钢板炉批号,并自动成册供船级社查验。
- **对本方案**:第11章归档导出 + 第14章 AI"提醒触发 NDT"而非替代 NDT。

### 1.7 离线优先,应对钢结构信号屏蔽

- **机理**:工人作业现场大量在双层底、分段内部、密闭舱室——钢板是天然的电磁屏蔽体,Wi-Fi/4G/5G 在里面基本"没有信号",不是"信号弱"。
- **❌ 不懂行**:把它写成"现场网络不稳定,支持离线缓存"一句话带过,默认在线为常态、离线为例外。
- **✅ 懂行**:offline-first 是一等公民——本地优先读写、断点续传、冲突合并、缓存加密;并对 5G 专网/UWB/信标做现实的覆盖评估,不假设全覆盖。
- **对本方案**:第6章把离线运行时作为横切层;第13章专门论证网络/终端现实。

### 1.8 对接真实船舶 CAD 设计系统

- **机理**:船厂的三维设计不是通用 CAD,而是 **AVEVA Marine、SPD(中船自主"船舶设计制造一体化")、TRIBON、NAPA** 等专业系统,它们的数据模型天生以分段/中间产品和船舶专业(船体/管系/电气/舾装)组织。
- **❌ 不懂行**:接口章节只写"三维建模工具:导入模型文件",连系统名都不点。
- **✅ 懂行**:点名真实设计系统,讲清结构树(分段)、零件/分段属性、坐标、材质、版本、图纸关联如何受控映射进本应用。
- **对本方案**:第12章明确以现场实际设计系统为准做集成(本稿先列候选,需现场确认),这是整个方案的技术命门。

### 1.9 设计变更与 as-built 闭环

- **机理**:造船周期以年计,设计变更(技术通知单/技术联络单/ECN)频繁;且现场实际常与图纸不符,需要把实测/实做(as-built)回灌,形成竣工模型与竣工资料。
- **❌ 不懂行**:只有"PDM 受控发布、移动端显示最新版"的单向下发,没有变更影响域,也没有现场反向回灌。
- **✅ 懂行**:变更能做影响域分析(波及哪些分段/工单/已完工内容)、双向闭环(现场 as-built 回到模型/资料)。
- **对本方案**:第12章变更接口 + 第13章精度/实测回灌共同支撑。

### 1.10 HSE 作业票(动火/受限空间/高空)

- **机理**:船厂现场是动火作业、密闭/受限空间作业、高空作业的高发地,**作业票(Permit to Work)**是安全合规的强制前置,且与工序强耦合(没票不准动火)。
- **❌ 不懂行**:安全只是工单上"安全要求"一栏文字提示。
- **✅ 懂行**:动火/受限空间/高空作业票数字化,与工序/工单联动(未取票则工序不可开工),含气体检测、监护人等要素。
- **对本方案**:第9章工单页含作业票状态;可作为"无纸化"里合规价值最直接的一块。

### 1.11 知识资产化,对抗 ETO 低复用

- **机理**:造船是按单设计制造(ETO),每条船都几乎重做,**复用率天然低**——这意味着"经验"是最易流失也最该被沉淀的资产,否则每条船都在重新踩坑、依赖老师傅个人记忆。
- **❌ 不懂行**:建个"知识库"挂在 AI 后面当配角,内容是静态规范 PDF 堆叠。
- **✅ 懂行**:把每次施工/返工/报验/退审结构化沉淀为可被 AI 调用的活资产,开工前主动推送给下一条船——知识越用越厚。
- **对本方案**:第10章知识工程与 AI 陪练,是本方案区别于"电子化表单"的灵魂。

### 1.12 服务船东/船级社的外部满意度

- **机理**:造船的"客户满意度"有一半不在厂内,而在**船东和船级社**手里——他们对一个船厂的评价取决于报验是否高效、资料是否齐全可追溯、整改是否闭环。这直接影响口碑与**复购(下一条船给不给你造)**。
- **❌ 不懂行**:把"客户"默认成厂内部门,船东/船级社只是抄送对象。
- **✅ 懂行**:把船东/船级社当作系统的一类正式用户,为其提供在线见证、资料一键调取、整改透明的体验。
- **对本方案**:第11章直接服务此目的,并呼应承研方的 CCS/船级社理解(第4.3)。

## 2. 造船行业核心痛点

> 双视角呈现:先按"干系人"(谁在痛),再按"建造阶段"(在哪痛)。撰写人若只会写"依赖纸质、人工签字",说明只看到表象;真正的痛点要收敛到根因。

### 2.1 痛点收敛:三个表象 → 一个根因 → 三个后果

- **表象**:① 流程靠纸和人传(图纸、工艺卡、报验单、签字流转慢)② 返工多、报验等待长,吃掉坞期 ③ 经验断层、新人上手慢。
- **根因**:**造船是"单件小批、强报验、经验密集"的项目型制造**——低复用让经验无法沉淀,强外部见证让流程被反复打断,密集协作让一次做对很难。
- **后果**:① 坞期/交期压力始终高悬 ② 质量与资料追溯依赖人,船级社/船东体验差 ③ 老师傅一退休,能力断档,无法快速复制到下一条船。

### 2.2 按干系人视角

| 干系人 | 核心痛点 |
|---|---|
| 一线工人 | 看不懂三维图;在分段内部没网络;干完活等报验;凭经验干、出错才知道 |
| 班组长/工段长 | 派工、找人、找料、协调多专业冲突全靠跑和喊;现场状态不透明 |
| 工艺员 | 工艺意图传递失真;同类问题反复被问;现场反馈回不到工艺改进 |
| 质检员/QC | 报验排队、重复跑现场;质量记录靠手填,成档耗时;追溯难 |
| 生产/计划主管 | 分段/车间/工序进度不实时;瓶颈(尤其坞期)看不清;物料齐套率不准 |
| 设计院所 | 变更下发到现场的"最后一公里"失控;现场 as-built 回不来 |
| 船东/船级社 | 见证点排期靠人盯;现场资料不齐、追溯慢;整改闭环不透明 |
| 管理层 | 经验无法资产化;新人培养周期长;无法把"造一条船的能力"标准化复制 |

### 2.3 按建造阶段视角(每阶段有各自独特的痛,绝非同一张流程模板)

| 阶段 | 该阶段独有的痛点(不能用统一模板糊弄) |
|---|---|
| 钢材加工 | 套料/切割(NC nesting)、零件标识与追溯、漏切/变形、套料利用率 |
| 小组立 | 缺漏件、装配方向、加强筋位置、点焊定位精度 |
| 中/大组立 | 多零件累积误差、焊接变形控制、胎架与基准 |
| 总段合拢 | 累积精度、余量决策、合拢口对中 |
| 船坞搭载 | **坞期瓶颈**、大节点吊装、精度合拢一次成功率、搭载网络 |
| 舾装安装 | 托盘齐套、区域多专业防碰撞、预舾装率、设备/管路/电缆走向、调试 |
| 涂装 | 表面处理等级、膜厚、环境窗口(温湿度)、工序占场与坞期争用 |
| 报验/见证(贯穿) | 三检+船东+船级社多层报验排队、见证点协调、退审与整改闭环 |

#### 2.3.1 钢材加工车间

- **独有痛点**:一条船数万个钢零件,**切下来后"谁是谁"难以识别**;套料(NC nesting)利用率直接影响钢材成本;切割热变形;坡口方向/漏切。
- **现状怎么糟**:纸质套料图;零件靠喷漆编号 + 老师傅记忆识别,喷码被打磨/锈蚀后下游"找件靠翻";套料利用率不透明;变形与返工记录在纸上。
- **数字化怎么解**:NC 套料数据贯通、零件**激光打标/二维码**、扫码识别与分拣配送、套料利用率看板;**AI 对焦可靠子集**——零件编号/切割轮廓识别、套料匹配、漏切提示。回扣尺子 #1/#6。

#### 2.3.2 小组立

- **独有痛点**:**缺漏件、装配方向装反、加强筋/肋板位置错**,高度依赖工人看图的准确性;点焊定位精度影响后道。
- **现状怎么糟**:纸质拼板图,工人凭经验看图;错装往往到下道工序甚至组立后才暴露,返工成本翻倍。
- **数字化怎么解**:三维 + 自动平面图看图、扫码确认零件、**AI 缺漏件/装配方向比对**(有模型 BOM 约束,较可靠)、班组长初审闭环。回扣尺子 #1/#11。

#### 2.3.3 中组立 / 大组立

- **独有痛点**:多个小组立合并,**误差累积**;**焊接变形控制**是核心难题;胎架与基准决定分段最终形状;大分段翻身/吊装。
- **现状怎么糟**:精度靠卷尺、塞尺人工抽测;变形靠老师傅经验"反变形"预留;数据记录在纸,无法回溯到下料环节定位误差源。
- **数字化怎么解**:**全站仪/三维扫描实测回灌**、精度数据闭环(可反查误差源)、焊接顺序工艺包推送、变形趋势预警。回扣尺子 #5/#6。

#### 2.3.4 总段合拢

- **独有痛点**:多分段合成总段,**累积精度**进一步放大;**余量割除决策**(割多了不可逆、割少了装不上);合拢口对中。
- **现状怎么糟**:余量决策依赖个别师傅经验,口口相传;一旦判断失误,要么坞内修割,要么返工。
- **数字化怎么解**:合拢前**精度预检**、基于实测的余量决策且全程可追溯、对中数据记录。回扣尺子 #5,直接服务 #2(坞期)。

#### 2.3.5 船坞搭载(全厂瓶颈)

- **独有痛点**:**坞期是全厂最贵资源**;大节点吊装资源协调;**精度合拢一次成功率**——装不上就现场修割,极费坞期;搭载网络(分段进坞顺序)牵一发动全身。
- **现状怎么糟**:搭载顺序靠计划员经验排;精度问题往往坞内才暴露;现场修割吃掉宝贵坞期;吊车/场地争用靠现场协调。
- **数字化怎么解**:**搭载前精度预检 + 缺漏件清零 + 报验前置**("进坞即可装、一次装对");搭载网络与坞期占用可视;远期接数字孪生做搭载/坞期仿真。回扣尺子 #2/#5,本方案 KPI 之首。

#### 2.3.6 舾装安装

- **独有痛点**:**区域内多专业打架**(管系/电气/暖通/铁舾在同一狭小舱室争空间与顺序);**托盘不齐套就停工**;预舾装率低则大量舾装被迫推到坞内/下水后做,又回头挤占坞期;设备/管路/电缆走向复杂;后续调试。
- **现状怎么糟**:各专业各自按图施工,碰撞到现场才发现、互相拆改;齐套靠人工核对清单;预舾装机会常被错过。
- **数字化怎么解**:工单**绑定托盘**、托盘齐套预警、**区域多专业冲突/施工顺序提示**、三维舾装模型现场看图、调试记录电子化。回扣尺子 #4。

#### 2.3.7 涂装

- **独有痛点**:**环境窗口受天气制约**(温度/湿度/露点不达标不能施工);表面处理等级(如 Sa2.5)与**膜厚(DFT)**检验;**PSPC**(压载舱涂层性能标准)合规记录繁重;占用场地/坞期且与其他工序争用。
- **现状怎么糟**:环境与膜厚靠手测手记;PSPC 资料厚且易缺漏;返锈/返工;环境窗口错过导致工期波动。
- **数字化怎么解**:环境(温湿度露点)与膜厚数据采集、PSPC 资料数字化自动成册、报验见证在线;环境窗口与排程联动提示。回扣尺子 #3/#12。

#### 2.3.8 报验 / 见证(贯穿全程)

- **独有痛点**:三检 + 船东 + 船级社**多层报验排队**;**验船师时间难约**,报验等待工时吃掉有效工时;退审整改不透明、反复跑。
- **现状怎么糟**:报验靠电话/纸单约;验船师到场前后资料东拼西凑;退审意见口头传达、整改无闭环记录。
- **数字化怎么解**:**见证点(Hold/Witness Point)排期、远程/电子见证、退审-整改-复验闭环、资料一键成册**。回扣尺子 #3/#6/#12,对应第11章。

> **小结**:8 个阶段,8 套不同的痛点与解法。原始方案把它们写成同一张"派工→看图→拍照→AI预审→审核"模板,正是"未深入现场"的证据。本方案在第9章按阶段差异化设计工单页与校验项。

## 3. 造船行业未来发展路径(本方案要踩在哪条趋势线上)

> 技术方案不能只解决今天,要让客户看到它站在行业演进的正确方向上。本方案的每项能力都对应下表的一条趋势。

| 趋势线 | 行业正在发生什么 | 本方案的落点 |
|---|---|---|
| **数字化 → 模型驱动(MBD/MBM)** | 从二维图纸驱动转向"模型即唯一权威数据源",基于模型制造 | 三维模型受控流转 + 三维转施工平面(第8章) |
| **经验数字化 → 知识资产化 → AI 陪练** | 老师傅经验从"口口相传"转为可被 AI 调用的结构化资产 | 知识工程 + 现场陪练(第10章) |
| **人工见证 → 远程/电子见证(e-survey)** | 船级社推进远程检验,船东远程见证成常态 | 在线见证与可追溯(第11章) |
| **有余量 → 无余量精度造船** | 三维测量(全站仪/激光扫描)闭环驱动精度决策 | 精度测量数据接入(第13章) |
| **自动化产线 / 机器人焊接** | 切割、焊接、涂装自动化产线与机器人渗透 | 接口预留,长远规划纳入(第17章) |
| **数字孪生 → 智能船厂** | 从看板可视化走向搭载/坞期仿真与智能船厂 | 长远规划(第17章) |
| **绿色/新能源船型** | 电动、双燃料(LNG/氨/甲醇)、新工艺与新质量要求涌现 | 知识库与工艺包需具备可扩展性 |
| **AI 多模态拐点** | 大模型/多模态使"看图、问工艺、查规范、比对资料"首次可落地于现场 | 本方案得以"AI 原生"的根本前提 |

## 4. 标杆案例与可借鉴的类似行业方案

> 分三层:① 造船业内直接对标(定方向)② 类似行业可平移的成熟做法(降风险)③ 承研方自有可复用资产(立信任)。

### 4.1 造船业内标杆(北极星)

> 逐个拆"做对了什么 → 我们平移什么"。具体经营数字不在此杜撰,仅取业内公认的能力特征。

**① 江南造船「数字江南」——模型驱动的全三维数字化造船**
- *做对了什么*:推进**全三维设计 + 基于模型的制造(MBD/MBM)**,把三维工艺、三维作业指导送到车间,推动**取消/减少二维图纸下厂**,以模型作为权威数据源贯穿设计-工艺-制造。
- *我们平移什么*:本方案"**模型驱动 + 三维转施工平面**"的对标基准(第8章)。但中远船厂当下更现实的路径是"模型 + 自动派生平面图并存",照顾一线读图习惯,逐步过渡到看模型——这是对江南范式的落地化裁剪。

**② 外高桥造船(SWS)——智能产线 + 超复杂产品数字化建造**
- *做对了什么*:钢加工/部件**智能产线**与车间级数字化;并在**首艘国产大型邮轮**这类**舾装量级极大、零件以百万计**的产品上,验证了海量中间产品/托盘的数字化组织能力。
- *我们平移什么*:**钢加工车间数字化**(第2.3.1、第9章)与**托盘/区域舾装**的数据组织(尺子#4、第12章)。邮轮的本质是"舾装地狱",恰好印证本方案把舾装多专业冲突/托盘齐套当重点是对的。

**③ 沪东中华(LNG 船)——极致焊接质量与可追溯**
- *做对了什么*:大型 LNG 运输船**殷瓦(Invar)薄膜围护系统**对焊接质量近乎苛刻,倒逼出业内最严的**焊工资质、WPS、检测与可追溯**体系。
- *我们平移什么*:**质量可追溯血缘链**(尺子#6、第11章)——焊工钢印↔WPS↔NDT↔材质炉批号一键反查的工程做法,是本方案归档能力的标杆。

**④ 现代重工(HHI)/ 三星重工(SHI)Smart Yard——数字孪生船厂**
- *做对了什么*:5G 智能船厂、**数字孪生**,其价值落点是**搭载网络仿真、坞期/资源优化、远程协同**,而非泛可视化看板。
- *我们平移什么*:为本方案**长远规划(第17章)**指明方向——数字孪生要服务坞期(尺子#2),与"搭载前精度预检"(第2.3.5)形成近-远呼应。

### 4.2 可借鉴的类似行业方案(同构问题、成熟做法)

**① 航空制造 MBD(COMAC C919 / Boeing)——模型即唯一权威数据源**
- *同构点*:超大型、强追溯、项目型;工人**看模型不看图**,工艺与检验信息(PMI)直接附在模型上。
- *平移什么*:"三维下厂"的成熟范式、**PMI/工艺信息叠加到模型**、模型轻量化按工位调用(第8章);以及"模型权威、图纸派生"的数据治理理念。

**② EPC / 工程建设(旁站监理 + 4D BIM)——强见证 + 海量资料 + 进度可视**
- *同构点*:与船厂"报验/见证 + 竣工资料归档 + 进度跟踪"几乎同构;监理旁站 ≈ 验船师见证。
- *平移什么*:**见证/旁站数字化、资料一键成册、4D 进度可视**(第11章 + 坞期可视),这是"客户满意度"那条线最直接的样板。

**③ 大型钢结构 / 海工平台 / 压力容器——焊接追溯 + NDT**
- *同构点*:项目型离散制造,焊接质量与无损检测是质量命脉。
- *平移什么*:**焊工/WPS/NDT/材质追溯链**的成熟工程做法(尺子#6、第11章),以及"AI 提示触发 NDT 而非替代 NDT"的可信边界(第14章)。

**④ 重型装备(项目型 MES)——单件小批、按单制造**
- *同构点*:与大批量汽车 MES 假设相反,与船厂一致。
- *平移什么*:**项目型 MES 的工单/状态/对象模型**(以项目-中间产品为核心,而非节拍线),印证第6章对象模型选型(尺子#1)。

### 4.3 承研方自有可复用资产(立信任,务实使用)

- 元启晟在船舶领域已具备 **CCS 原则性认可经验**与 **中船七一二所联合研制**背景,以及**中远体系既有合作关系**。
- 这些资产在技术方案中体现为:**对船级社规范与见证流程的理解(第11章)**、对船舶领域数据与合规要求的把握。
- 注:智慧驾驶舱属于"船舶运营/智能航运"域,与"造船制造执行"域不同,**本技术方案不引用该案例**,以保持技术聚焦与专业可信。

# 第二部分 · 总体技术方案

## 5. 总体目标与设计原则

### 5.1 总体目标

在不推翻船厂既有 PDM/CAPP/MES/ERP/APS 的前提下,建设一个 **AI 原生的无纸化制造现场应用 + 知识资产层**,实现"模型受控流转 → 一线移动施工 → AI 预审 → 人工复核 → 报验见证 → 全程归档"的闭环,并把四个目的(提效/知识复用/新人上手/客户满意)落到可度量的指标。

### 5.2 建设范围

- **在范围内(In Scope)**:现场制造执行域(派工/看图/施工/报工/质检/报验见证/整改/归档)、其上的 AI 能力层、知识资产层、移动作业端、与既有系统的受控接口、统一编码与现场数据治理。
- **不在范围内(Out of Scope,明确边界以控风险)**:不重写 PDM/CAPP/MES/ERP/APS 的核心功能;不主导自动化产线/机器人建设(只做接口预留);不替代 NDT 与人工质量判定;不承担船舶设计本身。

### 5.3 衡量成功的 KPI 体系(每项需与厂方共定基线)

| KPI | 定义 | 数据来源 | 对应目的 |
|---|---|---|---|
| 坞内一次合拢成功率 | 进坞分段无需现场修割即一次装配到位的比例 | 精度预检 + 搭载记录 | 提效/坞期 |
| 报验等待工时 | 报验申请到见证完成的平均等待工时 | 报验见证工作流时间戳 | 提效/客户满意 |
| 报验一次通过率 | 首次报验即通过(无退审)的比例 | 见证签认/退审记录 | 客户满意/质量 |
| 返工率 | 因施工错误导致返工的工单占比 | 工单/整改记录 | 提效/质量 |
| 新人独立上岗周期 | 新人从入职到可独立完成典型工序的周期 | 培养记录(对照组) | 新人上手 |
| 纸质单据流转下降率 | 试点/全厂范围内纸质图纸/工艺卡/报验单减少比例 | 业务统计 | 无纸化(手段) |
| 资料成册时长 | 报验/竣工资料从齐备到成册的耗时 | 归档系统 | 提效/客户满意 |

### 5.4 设计原则(每条都回扣第一部分的尺子)

1. **以中间产品/分段为对象建模**(尺子#1):全链路对象模型 = 船号→分段/总段→区域/托盘→工序→检验点。
2. **AI 原生,但边界清晰**(尺子#11、#6):AI 只做能稳定落地、且人可复核的事;不替代 NDT 与人工终审。
3. **离线优先**(尺子#7):移动端按"本地优先 + 后台回传 + 冲突合并"设计,不假设现场有网。
4. **模型驱动**(尺子#8):以受控三维模型为权威数据源,二维施工图为模型派生物。
5. **知识沉淀闭环**(尺子#11):每一次施工/报验/退审都喂回知识资产,越用越厚。
6. **薄前端 + 厚平台 + 可扩展能力**:现场入口轻,AI 与知识能力可独立演进。
7. **可度量**:每能力绑定 5.3 的 KPI,先建基线、后看改善。
8. **私有化与数据安全优先**:涉船舶图纸/工艺为敏感数据,优先私有化部署,数据不出厂(尺子相关,详见第14/15章)。

## 6. 总体技术架构(AI 原生分层)

### 6.1 分层架构(自上而下五层)

1. **入口层**:一线移动作业终端(三防平板/手机)、车间看板大屏、管理驾驶舱、**船东/船级社协同入口**(外部用户)。统一身份与权限贯穿。
2. **应用层**:工单与派工、三维/平面看图、扫码确认、拍照/视频报工、报验见证、整改闭环、电子签名、电子归档与一键成册。
3. **AI 能力层(本方案核心)**:多模态施工预审、知识问答/检索增强(RAG)、缺漏件/装配比对、资料完整性预审、新人现场陪练、异常提示。**以服务化方式被应用层调用,可独立迭代与评测**,模型可替换。
4. **知识与数据层**:知识资产库(工艺包/返工库/报验问题库/退审意见库/规范摘要/向量索引)、过程数据(施工证据/质量记录/状态流水)、**统一编码与对象模型(分段树)**、文件/模型对象存储。
5. **集成层**:与三维 CAD、PDM、CAPP、ERP、APS、MES 的受控接口(详见第12章),含接口适配、数据校验、版本控制。

### 6.2 横切能力(贯穿各层)

- **离线优先运行时**:本地数据库 + 同步引擎 + 冲突合并(第13章)。
- **统一审批流引擎**:串行/并行会签/条件分支/退回/转办/加签/抄送/超时提醒/版本冻结/审计(第15章)。
- **安全与权限**:RBAC + 数据权限 + 流程权限,水印/加密/审计(第15章)。

### 6.3 部署与技术取向

- **部署形态**:优先**私有化部署**(数据不出厂);AI 模型支持本地化/专有云部署,满足船厂保密要求。
- **架构取向**:微服务 + API 网关;能力服务化解耦,AI 与知识模块可独立升级;移动端跨平台、离线优先。
- **集成取向**:以**接口契约**对接既有系统,避免侵入式改造;数据以统一编码贯通。

## 8. 三维模型流转与工位级三维数字化作业(MBD)

> 本章承载本方案**最先落地**的能力——**基于轻量化三维模型的工位级数字化作业**。该能力**现已建成并在部分工位试运行**,是后续 AI 闭环、知识工程、报验见证的落地基座。

### 8.1 工位级三维数字化作业(已建成能力)

- **定义**:以三维模型替代二维图纸下发工位,即**基于模型的作业指导(Model-Based Work Instruction, MBWI / MBD 下厂)**。
- **能力形态(已落地 / 试运行)**:
  - 工位三防平板终端加载**轻量化三维动态模型**——可旋转、缩放、剖切、漫游的交互式三维,而非静态截图;
  - 系统由模型**自动派生施工视图**(正投影主视/俯视/侧视、剖切图、局部放大图)与**制造信息标注(PMI:尺寸、零件编号、工艺/焊接信息)**;
  - **只读可视化**:支持浏览、点击查询、属性查看、现场测量,**不可编辑修改**;
  - **轻量化 + 分块 + 按工序/分段过滤、按需加载**,适配现场终端与弱网/无网。
- **价值**:工人直接获取三维施工信息与精确尺寸,消除二维图纸转译与读图误差,支撑"去图纸化"与节拍化作业(对应"模型直达工位"诉求)。

### 8.2 模型受控流转路径

- 路径:**三维 CAD(正向设计模型)→ PDM 受控发布 → CAPP 施工包 → MES 工单 → 工位/移动端按工序加载**。
- 受控原则:终端**只显示 PDM 最新受控版本**,旧版自动失效,杜绝版本混乱(尺子#8/#9)。
- 模型成熟度:仅"受控/发布"状态模型可下厂(治理规则详见第16章设计域 SSOT/MBD)。

### 8.3 模型自动派生施工视图与标注的关键技术(逐项)

- **视图自动生成**:正投影主视/俯视/侧视、剖切图、局部放大图;按施工对象自动选取视角。
- **图面处理**:隐藏线消除、边界线提取、焊缝线提取。
- **标注叠加(PMI)**:尺寸自动标注、零件编号标注、工序信息叠加、质量检验点叠加、安全/作业票提示叠加。
- **过滤聚焦**:按工序/工位/分段过滤,只显本次施工对象,降低读图负担。

### 8.4 与一线协同的设计取向

- **三维动态模型为主体 + 自动派生施工视图/标注为辅助**:模型可交互浏览,关键工序自动出施工视图/标注辅助读图,新人与熟练工各取所需,平滑过渡(对江南"全模型下厂"范式的落地化裁剪)。
- **与工单绑定**:模型/派生视图与 MES 工单、检验点、托盘清单绑定,形成"一单一模一标准"。

## 9. 一线施工 AI 闭环(第一期英雄场景)

### 9.1 标准闭环

接单 → **三维模型/派生施工视图查看(工位级三维数字化作业基座见 8.1,已建成)** → 扫码确认零件/托盘 → 施工 → 拍照/视频报工 → **AI 预审(对焦可靠子集)** → 班组长初审 → 工艺/质检复核 → 报验/见证 → 整改闭环 → 电子归档。

### 9.2 移动端工单页应展示

工单号、船号、分段号、工序名称、施工对象、计划时间、工艺卡、三维模型、自动平面图、质量标准、安全要求(含动火/受限空间/高空**作业票状态**)、材料/**托盘**清单、工装设备、历史问题、审批流状态。

### 9.3 按建造阶段差异化(不复用单一模板)

- **钢材加工**:套料图/NC 数据、零件激光打标/扫码识别与分拣、套料利用率;AI 校验零件编号/切割轮廓/漏切。
- **组立(小/中/大)**:三维 + 平面看图、扫码确认零件、AI 缺漏件/装配方向比对、精度/变形记录。
- **船坞搭载**:搭载前精度预检 + 缺漏清零 + 报验前置;大节点吊装与搭载顺序提示。
- **舾装**:工单**绑定托盘**、托盘齐套预警、**区域多专业冲突/施工顺序提示**、调试记录。
- **涂装**:环境(温湿度露点)与膜厚采集、PSPC 资料自动成册。
> 即:阶段不同,工单页字段、校验项、AI 重点均不同(回扣 2.3 与尺子 #1/#4/#5)。

### 9.4 角色与职责(在闭环中的分工)

施工人员(接单/看图/施工/拍照/报工/报验)、班组长(派工/初审/整改确认)、工段长(协调/异常处理)、工艺员(工艺解释/符合性审核)、质检员(质量检查/签发结论)、生产主管(进度协调)、技术支持(设计/模型/工艺疑问)、船东/船级社(关键节点见证/抽检)、AI 助手(预审/问答/标准比对/异常提示)。

### 9.5 状态流转与整改闭环

- **状态机**:待派工→已派工→施工中→待预审→待初审→待复核→待报验→待整改→已完成,各状态留痕、可追溯。
- **整改闭环**:发现问题→开整改单→整改施工→复验→关闭,全过程记录照片/视频/意见/电子签名。
- **离线场景**:无网时本地完成接单/看图/拍照/报工,联网后自动回传与冲突合并(第13章)。

## 10. 知识复用与 AI 现场陪练(知识工程)

> (回扣尺子 #11)把"AI 知识问答"从搜索框升级为**现场陪练**——这是本方案区别于"电子化表单"的灵魂,直接服务"知识复用"与"新人快速上手"。

### 10.1 知识资产分类

- **规范标准类**:设计规范、工艺标准、WPS/PQR、质量检验标准、安全作业规程、设备安装手册、船级社规范摘要。
- **经验沉淀类(最稀缺)**:历史返工案例库、报验问题库、**船东/船级社退审意见库**、典型节点工艺包、老师傅作业要领。
- **过程数据类**:本船/历史船的施工证据、质量记录、变更记录,作为问答与陪练的上下文。

### 10.2 知识入库与治理闭环

- **沉淀**:每次施工/返工/报验/退审 → **结构化抽取**(关联船号/分段/工序/检验点/标准号)→ 入库并建向量索引。
- **治理**:知识有**责任人审核、版本、有效期、失效下线**;避免错误经验污染(知识也要"受控")。
- **冷启动**:首期以现有规范/工艺/标杆案例灌入,边用边沉淀经验类资产。

### 10.3 检索增强问答(RAG)与溯源

- **机制**:用户自然语言提问 → 检索知识库相关条目 → 大模型基于检索结果生成答案。
- **硬约束**:AI 回答**必须展示引用来源**(图号/工艺卡号/标准号/历史案例号),不可"裸答";无依据时回"无法判断"。

### 10.4 新人现场陪练场景

- 新人扫码看到的不只是工单,而是:"**这道工序老师傅怎么干、容易错在哪、上次为何被验船师打回、相关标准是什么**"。
- 边干边学、即时获得感,缩短独立上岗周期(对应 KPI),把老师傅经验从"个人记忆"转为"全员可调用资产"。

## 11. 船东/船级社在线见证与全程可追溯

> (回扣尺子 #3/#6/#12,呼应 EPC 旁站/4D BIM 类比)同时拉满"客户满意度"与"质量可追溯"的核心章节。

### 11.1 报验与见证工作流

- 流程:**报验申请 → 见证点(Hold/Witness Point)排期 → 现场/远程见证 → 签认 → (退审→整改→复验) → 关闭**。
- **Hold Point**(停工待检,未签不得继续)与 **Witness Point**(见证点)区分管理,与工序状态强联动。
- 多层报验:自检→互检→专检(厂 QC)→船东验船师→船级社验船师,逐层留痕。

### 11.2 远程/电子见证

- 支持验船师**远程视频见证**、电子签认,缓解"验船师时间难约、报验排队"痛点;对接船级社远程检验趋势(e-survey)。
- 见证过程录像、意见、签名完整留档。

### 11.3 全程可追溯血缘链

- 任一焊缝/构件可一键反查:**焊工资质/钢印 ↔ WPS/PQR ↔ NDT(RT/UT/MT/PT)报告 ↔ 钢板炉批号/材质单**。
- 血缘链作为数据模型固化,支撑随时追溯与质量分析。

### 11.4 资料一键成册

- 报验包/竣工资料按船级社/船东要求**自动归集成册**,成册时长从"天"级降到"小时"级(对应 KPI),整改透明可查。

## 12. 数据贯通与系统集成

> (回扣尺子 #8/#9)真实设计系统集成是本方案**技术命门**,须尽早确认。

### 12.1 接口对象与内容

| 接口对象 | 接口内容 |
|---|---|
| 三维 CAD(AVEVA Marine / SPD / TRIBON 等,**需现场确认**) | 模型文件、结构树(分段树)、零件/分段属性、坐标、材质、版本、图纸关联 |
| PDM | 图号/模型号、版本、审批状态、设计 BOM、变更单、发布状态 |
| CAPP | 工艺路线、工序卡、施工包、工艺 BOM、WPS、检验点、安全要求 |
| ERP | 物料需求、库存、采购、到货、成本、供应商;**按区域/托盘组织齐套**(尺子#4) |
| APS | 生产计划、排程、资源负荷、瓶颈(尤其坞期)预警 |
| MES | 工单、派工、报工、状态、质量结果、完工/异常记录 |

### 12.2 BOM 贯通与统一编码

- **EBOM → MBOM → PBOM 贯通**:设计 BOM 经工艺转换为制造/工序 BOM,落到分段/托盘可执行单元。
- **统一编码体系**:船号/分段/总段/区域/托盘/工单/图纸/模型/零件/物料/工序/检验点统一编码,是数据贯通的前提(需厂方主数据治理配合)。

### 12.3 集成方式与原则

- **非侵入**:以接口契约对接,不改造既有系统核心;通过 API/中间库/消息按数据特性选型。
- **数据校验与版本**:入站数据做完整性/一致性校验;模型与图纸严格版本控制,移动端只取最新受控版。
- **最小集成先行**:第一期先打通模型/工单/质量的最小闭环,作为是否扩大投入的关卡。

## 13. 现场工程现实(离线优先 / 网络 / 终端 / 精度)

> (回扣尺子 #5/#7)把"现场没网/钢结构屏蔽"当作架构一等公民,而非脚注。

### 13.1 离线优先架构

- **本地优先**:移动端本地数据库,接单/看图/扫码/拍照/报工**无网即可完成**。
- **同步与冲突**:联网后后台**断点续传**、增量同步、**冲突合并**(以服务端受控数据为准、现场证据不丢)。
- **安全**:离线缓存**加密**,图纸/模型禁止非授权导出(第15章)。

### 13.2 网络与定位现实评估

- 双层底/分段内部/密闭舱室**基本无信号**,不假设全覆盖;按区域做信号实测。
- 候选方案:**5G 专网 / Wi-Fi 6 / UWB / 蓝牙信标**组合,按区域与定位需求选型;关键是"无网也能干、有网即同步"。

### 13.3 终端选型

- **三防(防尘/防水/防摔)平板/工业终端**,适配焊接弧光、打磨粉尘、油污、潮湿环境;考虑续航、可佩戴/手套操作、扫码摄像质量。

### 13.4 精度测量接入

- 接入**全站仪 / 三维激光扫描**实测数据,回灌支撑余量/合拢精度决策(精度造船),服务"坞内一次装对"(尺子#5,支撑坞期)。

## 14. AI 能力的可信边界与工程化

> 这一章是技术可信度的关键,直接决定方案在船厂质检专家面前能否过关——也是规避"不懂行者过度承诺"的护栏。

### 14.1 能力边界(做什么 / 不做什么)

- **AI 做什么(可靠子集)**:套料/零件编号识别、缺漏件/错装比对、资料完整性与缺漏项预审、标准/工艺问答(RAG)、新人陪练、异常提示与"建议触发 NDT"。
- **AI 不做什么(明确不承诺,红线)**:**不**用现场普通照片判定焊接内部缺陷(咬边/气孔/裂纹/未熔合)——光照、油污、视角、锈蚀使其不可靠;焊接质量交给 **NDT(RT/UT/MT/PT)与人工**,AI 只做"提醒去做 NDT"。

### 14.2 结论分级与人机协同

- AI 结论四分级:**通过 / 疑似异常 / 不通过 / 无法判断**;**AI 永不终审,人工最终确认**。
- 误报宁可"疑似/无法判断",不放过(误放代价 > 误报代价)。

### 14.3 工程化与持续迭代

- **评测体系**:专用评测集、准确率/召回基线、误报代价分析。
- **上线方式**:灰度上线、可解释(展示判据/对比/置信度)、人在回路反馈再训练。
- **模型与数据安全**:优先**私有化/本地化部署**,敏感图纸工艺数据不出厂;模型可替换升级。

## 15. 安全、权限与合规

### 15.1 权限模型

- **RBAC + 数据权限 + 流程权限**三重控制。
- **数据权限**可按船号/分段/专业/车间/工序/审批节点细粒度控制;外部用户(船东/船级社)仅见授权范围。

### 15.2 数据与图纸防护

- 图纸/模型**加水印**(含人员/时间溯源)、移动端**禁非授权下载/导出**、离线缓存**加密**。
- 终端**设备绑定/MDM 管控**、实名登录、权限**到期自动回收**。

### 15.3 签名、审计与合规

- **电子签名/CA** 保证审批与见证的法律效力与不可抵赖。
- **全程日志审计**:操作留痕、版本冻结、可追溯。
- **审批流引擎**:串行/并行会签/条件分支/退回/转办/加签/抄送/超时提醒/版本冻结/审计。
- **合规取向**:满足等保要求与船厂保密制度;敏感数据私有化、不出厂。

## 16. 设计域:AI 辅助设计与数字化交付(全生命周期上游延伸)

> 本项目由船厂**技术中心(设计端)主导**,故在制造执行域(第5–15章)之外补强"设计域":向上延伸到设计与校审,向下延伸到数字化交付与运营,形成"设计→制造→交付→运营"全生命周期闭环。**本项目聚焦新造船,模型为正向设计产物。**

### 16.1 设计端"去图纸化"与单一权威数据源(SSOT / MBD)

- **MBD 原则**:以三维模型为**唯一权威数据源(Single Source of Truth)**,二维图纸为模型派生物;变更先改模型,派生件随动。
- **模型成熟度状态机**:在制(WIP)→ 校审中 → 受控 → 发布 → 作废;**仅受控/发布版可下游使用**,与第8章流转衔接。
- **模型数据标准**:几何 + **PMI 制造信息标注** + 结构树(分段)+ 统一编码(尺子#8)。
- **意义**:这是"去图纸化"能否成立的根——先有模型治理标准,后有工位去图纸化。

### 16.2 AI 辅助三维建模(明确边界,不过度承诺)

- **定位**:**AI 辅助、人工确认的半自动建模**,减少重复建模工作量;**非"一键全自动"**(回扣第14章 AI 可信边界)。
- **适用分级**:规则化结构件(型材、肋板、标准节点)优先;**复杂曲面外板、密集管系暂以人工为主、AI 辅助**。
- **流程**:二维设计信息/参数化输入 → AI 生成候选三维构件 → 设计人员校正确认 → 入受控库。
- **范围约束**:本项目限新造船(正向),不含修船/改装逆向扫描建模。

### 16.3 设计智能校审(设计端最可近期落地的 AI 价值点)

- **把"校审"拆为四类规则库**,分别建设、分别可计算:
  1. **合规性**:船级社规范、国标/国际公约等强制规则;
  2. **工艺性/可制造性(DFM)**:是否便于下料/装配/焊接/涂装,工厂工艺惯例;
  3. **一致性**:图模一致、专业间(船体/管/电/暖通)空间碰撞、BOM 一致;
  4. **经验性**:历史返工、退审意见沉淀的经验规则。
- **机制**:**规则引擎(确定性规则)+ AI(规范自然语言理解 / 模糊经验判断)**结合;结论分级(通过/疑似/不通过/需人工),**设计人员终审**。
- **价值**:降低人工校审成本、减少带病出图、**提升送审一次通过率**;与第10章知识工程、第14章 AI 边界一致。

### 16.4 设计变更数字化管理(三类变更闭环)

- **三类变更**:① **设计变更**(技术中心内部:技术通知单/技术联络单/ECN)② **船东变更**(商务:船东意见)③ **送审变更**(船级社退审/审图意见)。
- **闭环**:发起 → **影响域分析**(波及哪些模型/分段/工单/已完工/成本)→ 审批 → 受控发布 → 下发现场 → **as-built 回灌**(尺子#9)。
- **成本与结算**:记录变更的成本影响与依据,**避免后期结算争议**(对应会议诉求);与第12章变更接口贯通。

### 16.5 面向船东的数字化交付与运营延伸(全生命周期下游)

- **数字化交付物**:向船东交付**轻量化三维模型 + 结构化属性 + 关键资料**,支持备件追溯、维修指导。
- **交付标准**:对齐船级社/船东对**数字化交付(digital handover)/数字孪生交付**的要求。
- **运营延伸**:作为与公司**船舶运营/智能航运**业务的接口,可延伸至运维数字孪生(预留接口,不在本期承诺)。

### 16.6 船级社数字化送审(补强会议缺失项)

- 顺应**电子送审、模型送审、远程见证**趋势;与 16.3 的"合规性"校审规则库打通(送审前先做 AI 合规自检)。
- 送审记录、审图意见、退审整改纳入 16.4 变更闭环;技术中心是送审责任方。

> 回扣尺子 #3 / #8 / #9 / #11 / #12。本章与制造执行域(第5–15章)合,构成"设计→制造→交付→运营"全生命周期。

# 第三部分 · 实施与长远规划

## 17. 分阶段实施路径与长远规划

### 17.1 实施总则

1. **小步快跑、每步可验收、可随时止步**:每一期都有独立价值与量化验收;某期验收不达预期,可暂停或调整,不绑架后续投入。
2. **先深后广**:先在一个船号 / 一个车间把闭环跑透(深),再横向复制到全厂(广),避免一上来铺全厂、半年看不到效果。
3. **价值优先于覆盖**:排序依据是"对四个目的(提效/知识复用/新人/客户满意)和坞期的贡献",不是"系统功能清单的完整度"。
4. **AI 能力随数据成长**:AI 不是一次性交付,而是随知识资产与人在回路反馈持续增强(第14章);因此越早开始沉淀越好。
5. **诚实分层**:近期承诺可落地;中期有依赖、给条件;远期是愿景牵引、需单独可行性论证,不写成承诺。

### 17.2 总览

| 期 | 主题 | 周期(指示性) | 核心交付 | 风险/成熟度 |
|---|---|---|---|---|
| 第一期(近期) | 一线 AI 闭环 + 知识陪练 + 报验见证 | 试点 3–6 个月 | 单船号/单车间跑通闭环 | 低,可落地 |
| 第二期(中期) | 模型驱动铺开 + 精度闭环 + 托盘舾装 + 变更/as-built | 6–18 个月 | 全厂铺开 + 制造主线打通 | 中,有依赖 |
| 第三期(远期) | 多模态深化 + 产线接口 + 坞期数字孪生 + 智能船厂 | 18 个月+ | 愿景能力,逐项立项 | 高,需可行性论证 |

> 周期为指示性,实际取决于 17.6 的前置依赖落实情况。

### 17.3 第一期 · 近期(低风险、可落地)

- **目标**:在一个船号、一个车间(建议从**分段制造车间**或**舾装区域**择一)跑通"接单→看图→施工→AI 预审→人工复核→报验见证→归档"完整闭环,并启动知识资产沉淀。
- **范围**:一线施工 AI 闭环(第9章)+ 知识问答/新人陪练(第10章)+ 报验见证数字化与资料一键成册(第11章);移动端 + 离线优先(第13章)。
- **交付物**:移动作业端、按阶段差异化的工单页、三维转平面看图、AI 知识问答(带引用)、报验见证工作流、电子归档与一键成册、初版知识资产库。
- **量化验收(示例,需与厂方共定基线)**:
  - 试点范围内纸质图纸/工艺卡/报验单流转下降 ≥ 70%;
  - 报验等待工时下降 ≥ 30%;资料成册时间从"天"级降到"小时"级;
  - 一线工单移动端派发覆盖率 ≥ 95%,施工证据电子归档率 100%;
  - AI 知识问答采纳率、新人独立上岗周期(对照组)可度量改善。
- **前置**:确认真实三维设计系统并打通最小集成(第12章/17.6);完成试点区域网络/终端现实评估(第13章)。
- **退出机制**:验收不达标可止于试点,不进入第二期;沉淀的知识与数据资产仍归厂方所有、可继续使用。

### 17.4 第二期 · 中期(全厂铺开 + 制造主线打通,有依赖)

- **目标**:把闭环横向复制到全部建造阶段与车间,并打通对坞期最关键的三条制造主线。
- **四条工作线**:
  1. **模型驱动铺开**:三维下厂 + 自动平面图覆盖全阶段;模型轻量化与按工序加载工程化(第8章)。
  2. **精度测量闭环**:全站仪/三维扫描数据回灌,合拢前精度预检,服务"坞内一次装对"(第13章,尺子#5)。
  3. **托盘/区域舾装**:工单绑托盘、齐套预警、区域多专业冲突提示(第12章,尺子#4)。
  4. **设计变更与 as-built 闭环**:变更影响域分析 + 现场实测回灌竣工模型/资料(尺子#9)。
- **量化验收(示例)**:坞内一次合拢成功率提升、托盘齐套率与预舾装率提升、精度数据闭环覆盖率、变更下发到现场时效缩短——具体基线与目标值在第一期数据基础上确定。
- **依赖**:统一编码与数据标准治理需到位(第12章);与 ERP/APS/CAPP 的接口稳定;厂方流程配合调整。

### 17.5 第三期 · 远期(极具挑战的长远规划)

> 本期为**愿景牵引**,每项均需**单独立项 + 可行性论证**,不作为本方案的交付承诺。下面对每项坦诚说明"价值在哪、难在哪、本应用的角色"。

- **① 多模态施工预审深化**
  - *价值*:从"可靠子集"逐步扩展 AI 能覆盖的预审场景,进一步降低低级返工。
  - *挑战*:现场图像质量(光照/油污/视角/锈蚀)、标注数据获取成本高、误报代价大;需长期人在回路积累专用数据集。**焊接内部缺陷判定仍不交给照片 AI**(第14章红线不变)。
  - *角色*:持续演进的能力,而非一次交付;成熟度随数据成长。

- **② 机器人/自动化产线接口**
  - *价值*:与切割、焊接、涂装等自动化产线/机器人对接,实现工单-设备-质量数据贯通。
  - *挑战*:取决于**船厂自动化设备的投资节奏与接口开放度**(OPC UA / 设备协议);本应用不主导产线建设。
  - *角色*:做**接口预留与数据对接**,把自动化产线纳入同一数据底座。

- **③ 搭载/坞期数字孪生**
  - *价值*:对搭载网络与坞期做仿真与优化,直击全厂最贵瓶颈(尺子#2),与近期"搭载前精度预检"形成近-远呼应。
  - *挑战*:需要**高质量实测数据 + 可信排程模型 + 仿真引擎**三者齐备;建模复杂、对数据质量极敏感。
  - *角色*:近-中期沉淀的精度、状态、计划数据,是它的燃料;数字孪生是数据闭环的"收获期"。

- **④ 智能船厂(系统级愿景)**
  - *价值*:全厂级的智能化协同与决策。
  - *挑战*:这是跨多年、多投资、多供应商的系统工程,非单一应用可达。
  - *角色*:本方案作为**现场制造执行与知识资产的数据底座贡献者**,为智能船厂提供高质量数据与 AI 能力,而非声称"一套就是智能船厂"。

### 17.6 关键前置依赖(决定周期与成败)

1. **真实三维设计系统确认与集成**(AVEVA Marine / SPD / TRIBON 等):本方案技术命门,须尽早确认并打通最小集成。
2. **统一编码与数据标准治理**:船号/分段/区域/托盘/工单/图纸/模型/零件/物料/工序/检验点统一编码,需厂方主数据治理配合。
3. **现场网络与终端现实评估**:试点区域先做信号/定位实测与三防终端选型(第13章)。
4. **船东/船级社协同口径**:远程/电子见证的可行性与接口,需与对应船级社确认。
5. **流程与组织配合**:无纸化触及审批与岗位职责,需厂方制度与培训同步。

### 17.7 主要风险与应对(摘要)

| 风险 | 应对 |
|---|---|
| 设计系统集成难度被低估 | 第一期即做最小集成验证,作为是否扩大投入的关卡 |
| AI 过度承诺致信任崩塌 | 严守第14章可信边界,先做高可靠子集,人工终审 |
| 现场无网导致体验差 | offline-first 为架构一等公民,不依赖在线 |
| 数据标准不统一致贯通失败 | 统一编码作为前置治理,纳入第一期范围 |
| 一线抵触、习惯纸质 | 自动平面图照顾读图习惯;新人陪练带来即时获得感;试点先行树样板 |

### 17.8 小结

实施遵循"**先深后广、价值优先、诚实分层**":近期用低风险试点拿到可量化价值并沉淀数据资产;中期打通对坞期最关键的制造主线;远期以数据闭环为燃料,逐项立项迈向数字孪生与智能船厂。每一期都可独立验收、可随时止步——这既是工程稳健,也是对厂方投资负责。
